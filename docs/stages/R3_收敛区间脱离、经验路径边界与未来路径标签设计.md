# R3：收敛区间脱离、经验路径边界与未来路径标签设计

> 文档类别：Release / Future-Path Label Specification
> 所属阶段：R3
> 状态：草案 v0.3（按全量 exit-attempt 主样本、as-of 特征与标签可观测性重构）
> 前置依赖：R2 最终冻结的 canonical daily state、canonical event zone、event membership、状态版本登记册、最终 freeze manifest 与完整 lineage
> 后续阶段：R4 未来路径概率建模、序贯更新与解释研究
> 生效边界：R3 只定义冻结收敛事件中每一次合法 natural exit attempt、未来路径参考边界、五类主路径标签、技术性非主类状态、`T0/T+1/T+2` landmark 信息集、在线可打分风险集、标签可观测性与历史建模资格、删失规则和 canonical 标签产物；不修改 R2 冻结的 `W/q/K/d/g`、PCVT 指标、状态版本或 event-zone 规则，不拟合最终预测模型，不以模型准确率、未来收益、回测结果或交易利润反向选择状态定义、路径边界和标签规则。

---

## 0. v0.3 关键修订

v0.3 基于对 R3 分析单位、Landmark 时点语义和监督目标资格的进一步审查，完成三项结构性修订：

1. **分析单位修订**：每一次从 `confirmed_state=true` 发生的合法 `natural_state_exit` 都生成正式 `exit_attempt` 并进入 R3 主数据集；不再只把每个 `event_id` 的第一次退出作为主样本，也不再把后续退出降级为 sidecar。
2. **特征时点修订**：删除有前视歧义的“首先失效的维度”，改为严格受 `feature_available_time <= landmark_time` 约束的 `exit_trigger_dimension_set_at_T0`、`failed_dimension_set_as_of_landmark` 和各维度 `first_observed_failure_time_as_of_landmark`。
3. **风险集修订**：将“当时能否生成预测”的 `landmark_scoring_eligible` 与“历史上能否用于某个 horizon 的监督训练/验证”的 `target_observable_H`、`model_training_eligible_H`、`model_validation_eligible_H` 分开。右删失或标签为 `NULL` 的事件可以保留在线可打分记录，但不得进入对应 horizon 的监督目标集。

上述修订不改变五类主路径本体。`RECONVERGENCE` 仍是一次 exit attempt 的合法终止结果；重新收敛后再次发生合法退出时，生成新的 `exit_attempt_id` 并重新研究其未来路径。

## 1. R3 阶段目标

R3 的目标是把 R2 已冻结的收敛事件转化为可审计、可重复、无前视的 exit-attempt 级未来路径研究样本，并建立一套明确回答“股票每一次从已确认的收敛状态中退出时，从这个退出时点开始，未来最终走成什么路径”的标签语言。

R3 不以人为规定“放量一倍”“突破一个 ATR”或“涨跌超过某个百分比”作为有效突破定义。R3 先使用历史对照路径建立普通波动的经验参考边界，再依据价格路径穿越边界的顺序、是否重新进入冻结收敛状态以及路径可观测性，形成五类主路径和技术性非主类状态。

R3 需要回答七个问题：

1. **何时开始观察每一次脱离之后的走势**：对一个已经取得 R2 事件资格的收敛事件，每一次合法 `confirmed active → natural exit` 转换如何生成独立 `exit_attempt_id`，并把该退出日定义为该 attempt 的 `T0`。
2. **`g` 与多次 T0 如何分工**：`g` 继续负责 R2 event-zone 身份和 component 合并；R3 在 event 最终关闭前仍记录每一次合法 natural exit，并用 `event_id`、`exit_attempt_ordinal` 和 `component_count_as_of_exit` 保留其生命周期位置。
3. **什么叫有意义的向上或向下移动**：如何从匹配对照或普通时期的经验路径分布中建立随 horizon 变化的上下参考边界，而不是使用固定魔法值。
4. **五类主路径如何量化**：如何把向上、先下后上、向下、先上后下、重新收敛转化为确定性、可逐事件重放的标签规则。
5. **不能归入五类的样本如何处理**：如何区分无清晰路径、日频顺序不可识别、多次转折、异常终止、右截尾和有效观测不足。
6. **观察窗口如何设置**：如何使用 `H={5,10,20,30}` 区分早期反应、短期形成、主路径和延伸稳定性，避免把所有 horizon 视为同等主假设。
7. **如何支持后续动态判断**：如何为 `T0/T+1/T+2` 生成严格时点一致的 landmark 特征快照，使 R4 能在信息充分时立即判断，在信息不足时延迟一天或两天更新概率；同时如何区分在线可打分资格与对应 H5/H10/H20/H30 标签是否完整可观测。

R3 的正式结论只能表述为事件、边界和标签规则，例如：

```text
对每个冻结收敛事件，已经枚举其全部合法 natural exit attempts，并为每个 attempt 定义独立 T0；
已经使用设计样本中的匹配对照路径冻结上下经验边界；
已经按边界穿越顺序和重新收敛规则生成 H5/H10/H20/H30 路径标签；
已经形成 T0/T1/T2 landmark 特征及其风险集。
```

R3 不得表述为：

```text
某类 T0 形态必然上涨；
放量达到某个倍数就是真突破；
某个路径边界具有交易最优性；
T+1 或 T+2 一定比 T0 更好；
某个模型已经具有稳定交易优势；
R3 的未来路径结果可以反向修改 R2 状态定义。
```

## 2. 阶段定位与研究边界

研究主线如下：

```text
D0 → D1 → D2 → D3
               ↓
              R0 → R1 → R2 → R3 → R4 → R5 → R6
```

R2 已经回答：

```text
什么是冻结收敛状态；
哪一天 confirmed_state=true；
confirmed interval 是否达到 d qualification；
相邻 qualified components 是否按 g 归入同一 event zone；
event_id、zone revision、finalization、删失和可用时点如何记录。
```

R3 不重新设计收敛区间。R3 从已经取得资格的冻结收敛事件出发，研究：

```text
每一次合法 natural exit 发生在哪一天；
脱离后未来价格路径是否超出普通波动；
路径先向上、先向下、反转还是重新收敛；
哪些样本无法可靠归类；
截至 T0、T1、T2 分别可以观察到哪些特征。
```

R4 才负责：

```text
五类路径的频率、幅度、持续期和条件分布；
T0/T1/T2 多分类概率模型；
拒判与延迟更新；
概率校准、判断覆盖率和高置信度错误率；
PCVT 四维 score、八项子指标、量价和环境特征的解释力；
状态 1 / 状态 2、strict core / shell 的比较。
```

R5 才负责真实成交时点、成交概率、成本、止盈止损、持有期、组合约束和样本外交易验证。R3 可以保存 `earliest_trade_time` 等旁路字段，但不得用交易收益定义未来路径。

## 3. 当前正式研究对象

R3 只消费 R2 final freeze manifest 登记的冻结版本。当前研究对象为两条独立状态线：

```text
状态 1：
state_line = S_PCT
window_track_id = W120
W = 120
K = 3
qP = .20
qC = .20
qT = .25
qV = .20
d = 2
g = 1
strict core：同 W/K 下 qT = .20

状态 2：
state_line = S_PCVT
window_track_id = W120
W = 120
K = 3
qP = .20
qC = .20
qT = .20
qV = .30
d = 2
g = 1
strict core：同 W/K 下 qV = .20
```

两条状态线必须保持不同的 `state_version_id`、`event_id` 和结果表身份。二者不仅在 V 条件上不同，T/V 阈值也不同，因此 R4 可以比较两个冻结产品的实际路径差异，但不得把差异直接解释为 V 的纯增量因果效应。

## 4. R2 状态层级与 R3 消费边界

### 4.1 `K/d/g` 的角色

R2 的执行顺序保持不变：

```text
raw_state
→ K confirmation
→ confirmed_state
→ atomic confirmed interval
→ d qualification
→ qualified component
→ g grouping
→ event zone
```

三者的职责分别为：

```text
K：决定某一天何时成为 confirmed_state=true；
d：决定一段 confirmed interval 是否持续足够久、取得 component 资格；
g：决定两个 qualified components 是否归入同一个 event zone。
```

R3 不重新解释或修改上述规则。

### 4.2 R3 不等待 event zone 最终关闭才记录 T0

`g` 解决的是 event-zone 身份治理，不解决未来路径的观察起点。一个 qualified event 每一次从 confirmed active 转为自然退出时，R3 都应立即记录新的 `exit_attempt`，即使 R2 event zone 仍处于 `GAP_PENDING`，后续可能重新资格化并合并回原 `event_id`。同一 event 后续重新 confirmed 并再次自然退出时，必须生成新的 `exit_attempt_id`，不得覆盖或合并此前 attempt。

因此：

```text
T0 是否存在
≠ event zone 是否已经 finalized
≠ 后续是否重新并回原 event_id
```

后续同一 event 的结构结果作为 T0 之后的事实保存，例如：

```text
SAME_ZONE_REQUALIFIED
G_EXCEEDED_FINALIZED
UNQUALIFIED_REENTRY
QUALITY_BREAK
RIGHT_CENSORED
```

这些字段用于解释路径和审计 event 身份，但不撤销已经真实发生的 T0。

### 4.3 示例

假设一个事件已经取得 `d=2` 资格，并持续到第 5 日：

```text
第1日 raw=true
第2日 raw=true
第3日 confirmed=true，confirmed interval 起点
第4日 confirmed=true，达到 d=2，event 取得资格
第5日 confirmed=true，qualified active
第6日 raw=false
```

则：

```text
第6日 = T0
第6日记录 natural exit_attempt
g_used 按 R2 规则更新
event zone 可继续 GAP_PENDING
```

不论第 7 日以后是继续远离、重新 raw=true、重新 confirmed、重新 qualification，还是 `g` 超限，R3 都从第 6 日开始观察未来路径。

## 5. 分析单位与 T0 定义

### 5.1 event-zone 主单位

每个 R2 `event_id` 是一条冻结收敛事件。event-zone 表用于保存：

```text
state_version_id
event_id
security_id
first_qualification_time
first_component_start_date
last_confirmed_end_date
zone_finalization_time
zone_status
component_count
bridge_count
strict_core composition
```

### 5.2 exit-attempt 子单位

一个 event zone 内可能存在多次 confirmed active → natural exit 转换。每次转换形成一个 `exit_attempt_id`。

一次合法 natural exit attempt 至少满足：

```text
对应 state_version 已经存在 qualified event；
上一有效状态处于 confirmed active；
当前 expected trading row 存在；
eligible = true；
quality_state = valid；
raw_state = false；
confirmed_state = false；
exit_reason = natural_state_exit。
```

当前日定义为：

```text
e_{ij} = T0_{ij} = exit_attempt_time
```

下列情况不得生成正常 T0：

```text
quality_interruption；
unknown / blocked / diagnostic_required；
missing expected row；
ineligible；
sample-end censoring；
尚未形成 qualified event 的普通 raw=false。
```

### 5.3 全量 exit-attempt 主样本

R3 的正式主分析单位为：

```text
state_version_id × event_id × exit_attempt_id
```

每一次合法 `confirmed_state=true → natural_state_exit` 转换均生成正式 `exit_attempt_id` 并进入主数据集。不得只保留每个 event 的第一次退出，也不得把同一 event 的后续退出降级为 sidecar。

核心研究问题固定为：

> 股票每一次从已确认的收敛状态中退出时，从这个退出时点开始，未来最终走成什么路径？

同一 event 内可以依次出现：

```text
attempt 1 → RECONVERGENCE
attempt 2 → UP_THEN_DOWN
attempt 3 → UP
```

这不是重复或标签冲突，而是同一收敛生命周期内多次真实判断机会。`g` 的意义在于说明这些 attempts 是否属于同一个更大的 event zone，而不是取消第二次、第三次退出的研究资格。

每条 attempt 至少记录：

```text
exit_attempt_ordinal
source_component_id
source_component_ordinal
component_count_as_of_exit
zone_revision_as_of_exit
final_component_count
```

其中：

```text
exit_attempt_ordinal
source_component_ordinal
component_count_as_of_exit
zone_revision_as_of_exit
```

必须是 T0 当时可知字段，可以进入 R4 特征候选；`final_component_count` 是事后字段，只能用于描述和分层审计，不得进入 T0/T1/T2 预测特征。

### 5.4 多次 attempts 的相关性治理

同一 event 内的多次 attempts 都进入主数据集，但不得当作完全独立同分布样本。正式治理要求为：

```text
同一 event_id 的所有 attempts 必须进入同一个 sample split；
不得把 attempt 1 放入训练集、attempt 2 放入验证集；
重采样、置信区间和显著性推断至少按 event_id 聚类；
更稳健的推断可按 security_id 聚类；
未来路径窗口重叠必须显式标记；
```

R4 应同时报告两种口径：

```text
attempt-weighted：
每一次退出机会权重相同，作为主口径；

event-balanced：
每个 event 总权重为 1，内部 attempts 平分权重，作为稳健性口径。
```

若两种口径结论差异显著，说明反复退出 event 对总体结果具有 material 影响，必须独立解释，不得只报告更有利的口径。

## 6. 时间语义与观察窗口

### 6.1 收敛资格时点

对事件 `i`：

```text
c_i = first_qualification_time
```

它表示该收敛事件在实时意义上首次取得 R2 事件资格的时点。以下字段另行保留：

```text
raw_run_start_date
confirmed_start_date
first_qualification_time
```

三者不得混用。

### 6.2 每次脱离 attempt 的 T0

```text
e_{ij} = T0_{ij} = the j-th valid natural exit attempt within qualified event i
```

每个 `exit_attempt_id` 都有自己的 T0；该 T0 是该 attempt 未来完整路径和 Landmark 信息体系的统一锚点。

### 6.3 Landmark 信息时点

对每个合法 T0：

```text
T0 = e_i
T1 = T0 后第 1 个有效、可观测交易日
T2 = T0 后第 2 个有效、可观测交易日
```

有效日必须满足：

```text
expected trading row 存在；
eligible = true；
quality_state = valid；
相关特征在当时已经可得。
```

停牌、缺失、质量中断不能静默计为正常 landmark 日。若无法形成 T1/T2，必须记录删失或不可观测原因。

### 6.4 路径观察窗口

正式 horizon 为：

```text
H ∈ {5, 10, 20, 30} 个有效交易日
```

其角色固定为：

```text
H5：早期反应窗口；
H10：短期路径形成窗口；
H20：primary path-classification window；
H30：延伸稳定性和晚期反转检查。
```

H5/H10/H20/H30 不是四个地位相同的主终点。R3 以 H20 作为主路径标签，H5/H10 描述路径早期形成，H30 检查 H20 标签是否在更长窗口内反转或失效。

R3 必须生成 horizon 转移表：

```text
H5 → H10
H10 → H20
H20 → H30
```

T0/T1/T2 是信息截止时点；H5/H10/H20/H30 是未来结果观察窗口。二者不得混淆。

## 7. 为什么不使用固定技术指标阈值定义有效突破

“放量一倍”“越过区间上沿 3%”“超过 1 个 ATR”可以是事前特征，但不应直接成为未来路径的定义，原因包括：

1. 不同证券、年份、波动环境和流动性条件的尺度不同，固定阈值缺少可比性。
2. 剧烈移动可能随后反转，轻微移动也可能逐步形成持续方向。
3. 如果用成交量定义“有效突破”，再用成交量预测该标签，会形成循环论证。
4. 事后根据哪一个阈值产生最好收益来选择，会把 R3 标签设计变成交易参数搜索。

因此 R3 采用：

```text
未来价格与状态路径定义标签；
T0/T1/T2 的 PCVT、量价和环境变量用于 R4 预测标签。
```

## 8. 经验参考路径与有效移动边界

### 8.1 基本思想

R3 不先规定某个固定涨跌幅，而是回答：

> 在没有特殊冻结收敛脱离事件的普通条件下，同类股票在未来第 h 日通常会移动多远？

对每个 horizon day `h`，使用设计样本中的匹配对照路径估计普通移动分布，并形成：

```text
B_plus(h)  = 上方经验有效边界
B_minus(h) = 下方经验有效边界
```

边界可以随 h 变化，避免把第 1 日和第 20 日使用同一固定尺度。

### 8.2 对照风险集

每个 T0 的对照候选至少要求：

```text
同一交易日或允许的紧邻市场时点；
证券和数据状态有效；
不处于对应冻结收敛状态及其 exit attempt；
未来路径可按同一规则观察；
不使用未来信息筛选；
具有可匹配的 strict-past 波动、流动性、规模、行业和市场环境信息。
```

匹配变量可以包括：

```text
事件前波动尺度；
市值和流动性；
行业或市场分组；
过去收益和价格位置；
市场指数与行业环境；
上市年龄及可观测性。
```

对照设计的目的不是在 R3 证明因果效应，而是校准“普通噪声会移动到什么程度”。

### 8.3 标准化路径

对事件 `i`：

```text
y_i(h) = log(P_i(T0+h) / P_i(T0)) / s_i(T0-)
```

其中 `s_i(T0-)` 必须只使用 T0 以前的信息，例如 strict-past robust volatility、ATR/price 或冻结的其他稳健尺度。

R3-T02 必须在正式标签生成前冻结主标准化尺度。候选尺度只能依据：

```text
跨证券可比性；
缺失率；
年份稳定性；
极端值稳健性；
与 strict-past 语义一致性；
```

进行选择，不得依据哪一种尺度使状态事件收益最好或路径最容易预测。

### 8.4 经验边界估计

可采用匹配对照标准化累计路径的经验分位或共形校准：

```text
B_plus(h)  = upper calibrated control boundary at h
B_minus(h) = lower calibrated control boundary at h
```

具体误差容忍水平、分层与最低样本量由 R3-T02 在看到正式状态路径标签前预注册。该数值表达允许的普通噪声误标率，是统计错误政策，不得按未来收益优化。

若某个分层样本不足，必须按预注册层级进行 pooling 或返回 boundary unavailable，禁止临时放宽门槛以生成更多标签。

### 8.5 边界版本治理

每套边界必须绑定：

```text
boundary_version
training/design sample period
control definition
matching specification
standardization scale
error tolerance
horizon day
stratum definition
sample count
boundary estimate
uncertainty / calibration diagnostics
```

边界只能在设计样本学习。验证样本和样本外样本只能读取已冻结边界，不得重新估计。

## 9. 五类主路径的量化定义

### 9.1 预计算字段

对每个 T0、每个 horizon H，计算：

```text
standardized cumulative path y(h), h=1...H
upper_hit_day  = 首次 y(h) >= B_plus(h) 的有效日
lower_hit_day  = 首次 y(h) <= B_minus(h) 的有效日
upper_hit_count
lower_hit_count
boundary_crossing_sequence
reconvergence_qualification_day
same_zone_requalification_day
terminal_standardized_return
turning_point_count
```

上下边界穿越使用同一版本的日频可观测价格字段。若同一天 high/low 同时跨越上下边界，而日频数据不能识别先后顺序，必须进入技术性状态，不得使用收盘价猜测盘中顺序。

### 9.2 `RECONVERGENCE`

主规则为：

```text
T0 以后，在首次有效上边界或下边界穿越之前，
对应冻结 state_version 重新形成合法收敛事件资格，
即重新达到 K confirmation 并满足冻结 d qualification
→ 当前 exit attempt 标记为 RECONVERGENCE
```

该定义使 `RECONVERGENCE` 成为一次 exit attempt 的完整终止结果，而不是对同一 event 后续全部走势的永久归类。重新收敛后若再次发生合法 natural exit，必须生成新的 `exit_attempt_id`，并从新的 T0 重新研究未来路径。

R2 的结构身份进一步保存为：

```text
REENTERED_SAME_ZONE_REQUALIFIED
RECONVERGED_AS_NEW_EVENT
```

若仅重新达到 K confirmation、但在达到 d qualification 前再次退出，则记录：

```text
RECONFIRMED_BUT_UNQUALIFIED_REENTRY
```

该状态不自动赋值为 `RECONVERGENCE`，仍按当前 attempt 的后续方向、技术状态或观察窗口终止规则处理。

若有效方向边界已先被穿越，之后才重新取得事件资格，则主路径按方向穿越顺序分类，并另记：

```text
reconvergence_after_directional_move = true
```

### 9.3 `UP`

满足：

```text
upper_hit_day 存在；
lower_hit_day 不存在，或在 H 内未出现有效反向下边界穿越；
不存在优先级更高的 pre-break RECONVERGENCE；
路径不存在超过技术门槛的复杂多次主转折。
```

它表示 T0 后形成了超出普通噪声的主要向上移动，且观察窗口内未形成有效向下反转。

### 9.4 `DOWN`

满足：

```text
lower_hit_day 存在；
upper_hit_day 不存在，或在 H 内未出现有效反向上边界穿越；
不存在优先级更高的 pre-break RECONVERGENCE；
路径不存在超过技术门槛的复杂多次主转折。
```

### 9.5 `DOWN_THEN_UP`

满足：

```text
lower_hit_day < upper_hit_day；
先有效穿越下边界，随后有效穿越上边界；
不存在更早的 pre-break RECONVERGENCE；
在第二次方向穿越后，没有形成无法由单次主要反转解释的复杂多转折。
```

这对应“先下后上”，但不预先把第一次下行解释为假摔；“假摔”属于机制解释，只有路径事实本身被直接定义。

### 9.6 `UP_THEN_DOWN`

满足：

```text
upper_hit_day < lower_hit_day；
先有效穿越上边界，随后有效穿越下边界；
不存在更早的 pre-break RECONVERGENCE；
在第二次方向穿越后，没有形成无法由单次主要反转解释的复杂多转折。
```

“诱多”“骗炮”等属于后续解释，不写入正式标签名称。

### 9.7 标签优先级

正式判定顺序固定为：

```text
1. 检查异常、可观测性和右截尾；
2. 检查同日上下边界顺序不可识别；
3. 检查首次方向边界之前是否重新 confirmed；
4. 检查上下边界首次穿越顺序；
5. 检查是否存在复杂多次主转折；
6. 未满足五类规则则进入技术性非主类状态。
```

不得因某类样本数量少而在正式运行后临时改变优先级。

## 10. 技术性非主类状态

`path_class` 与 `label_status` 必须分开：

```text
path_class:
    UP
    DOWN_THEN_UP
    DOWN
    UP_THEN_DOWN
    RECONVERGENCE
    NULL

label_status:
    ASSIGNED
    NO_CLEAR_PATH
    AMBIGUOUS_SAME_DAY_ORDER
    COMPLEX_MULTI_TURN
    RIGHT_CENSORED
    INSUFFICIENT_OBSERVABILITY
    ABNORMAL_TERMINATION
    LABEL_CONTRACT_FAILURE
```

### 10.1 `NO_CLEAR_PATH`

未来窗口数据完整、无异常，但：

```text
未穿越上边界；
未穿越下边界；
未重新 confirmed；
```

或路径变化始终处于普通噪声区域。

它是正常经济结果，不是数据错误。R4 应报告其比例，不能从样本中删除。

### 10.2 `AMBIGUOUS_SAME_DAY_ORDER`

同一日 high/low 同时触及上下边界，日频数据无法确认先上后下还是先下后上。不得根据收盘方向、后续结果或主观解释补写顺序。

### 10.3 `COMPLEX_MULTI_TURN`

路径在 H 内出现超过一次主要方向反转，无法由 `DOWN_THEN_UP` 或 `UP_THEN_DOWN` 的单次转折解释。具体主要转折识别和稳定性门槛必须在 R3-T03 预注册，并由合成路径测试验证。

### 10.4 `RIGHT_CENSORED`

样本自然结束导致未来有效交易日不足 H。不得用较短可见路径冒充完整 H 日标签。

### 10.5 `INSUFFICIENT_OBSERVABILITY`

未来窗口仍在样本范围内，但因停牌、长期缺失或不可观测状态，未达到预注册最低有效观测要求。

### 10.6 `ABNORMAL_TERMINATION`

包括：

```text
无法归因的公司行为机械路径；
质量 hard break；
退市或长期不可观测；
关键价格字段失效；
其他预注册异常终止。
```

涨停或跌停本身是真实市场路径，不自动归入异常。只有数据或路径无法合法解释时才标记异常。

### 10.7 `LABEL_CONTRACT_FAILURE`

用于机器规则出现不可能状态、互斥标签同时成立、边界版本缺失或 lineage 无法闭合。出现该状态必须停止下游推进，不能把它当作普通未分类样本。

## 11. H5/H10/H20/H30 路径体系

每个 horizon 独立按同一标签规则生成：

```text
path_class_H5
path_class_H10
path_class_H20
path_class_H30
```

主终点为：

```text
primary_path_class = path_class_H20
```

H5/H10 用于观察早期路径形成，H30 用于检查 H20 路径的延续、反转或重新收敛。

必须输出：

```text
H5→H10 transition matrix
H10→H20 transition matrix
H20→H30 transition matrix
```

典型变化如：

```text
H5=UP → H20=UP_THEN_DOWN
H10=NO_CLEAR → H20=DOWN_THEN_UP
H20=UP → H30=UP
```

这些变化是研究事实，不得为了让标签更稳定而事后修改 H20 主窗口。

## 12. Landmark 特征体系

### 12.1 使用场景

Landmark 特征只用于存在动态更新需求的场景：

```text
T0 信息足够 → R4 可立即形成路径概率判断；
T0 信息不足 → 进入 T1 risk set，加入一天新增信息；
T1 仍不足 → 进入 T2 risk set，再加入一天新增信息；
T2 仍不足 → unresolved，不强行判断。
```

Landmark 不用于定义历史路径标签本身。五类标签只依赖未来价格/状态路径、经验边界和可观测性规则。

### 12.2 在线可打分风险集与历史监督目标资格

R3 必须把“当时能否生成预测”和“事后能否用于某个 horizon 的训练或验证”拆成不同资格字段。

#### 在线可打分风险集

```text
T0 landmark_scoring_eligible：
    全部合法 exit attempts；
    T0 特征截至 T0 可得；
    未发生异常终止。

T1 landmark_scoring_eligible：
    T0 被 R4 延迟为未决；
    T1 有效且可观测；
    T1 特征截至 T1 可得；
    截至 T1 未异常终止。

T2 landmark_scoring_eligible：
    T1 仍被 R4 延迟为未决；
    T2 有效且可观测；
    T2 特征截至 T2 可得；
    截至 T2 未异常终止。
```

是否在 T0/T1 被判定为“未决”由 R4 预注册的概率可靠性、预测集合或拒判规则决定。R3 只冻结结构、时点和基础可用性字段。

#### 标签可观测性

对每个 horizon `H ∈ {5,10,20,30}`，必须独立生成：

```text
target_observable_H
label_status_H
path_class_H
target_end_time_H
target_observation_count_H
```

其中：

```text
target_observable_H = true
```

至少要求：

```text
对应 attempt 的 H 日标签完整可重放；
path_class_H 非 NULL；
label_status_H 属于允许建模的结果状态；
不是 RIGHT_CENSORED；
不是 INSUFFICIENT_OBSERVABILITY；
不是 ABNORMAL_TERMINATION；
不是 LABEL_CONTRACT_FAILURE。
```

右删失或标签不可观测的 attempt 可以保留 `landmark_scoring_eligible=true`，因为真实运行中当时仍可生成预测；但不得进入对应 horizon 的监督训练、调参、准确率、校准或模型验证分母。

#### 历史模型资格

```text
model_training_eligible_H
=
landmark_scoring_eligible
AND target_observable_H
AND sample_split_role = model_development

model_validation_eligible_H
=
landmark_scoring_eligible
AND target_observable_H
AND sample_split_role = model_validation
```

不得用一个笼统的 `risk_set_eligible` 同时表达在线打分和历史监督学习资格。

### 12.3 允许的特征家族

R3 应生成而不预先保证有效的候选特征：

#### PCVT 结构特征

```text
P/C/T/V 四维连续 score；
四维 score 与冻结阈值的距离；
四维 score 的 T-1→T0、T0→T1、T1→T2 变化；
八项底层指标的 strict-past 标准化 score；
exit_trigger_dimension_set_at_T0；
failed_dimension_set_as_of_landmark；
newly_failed_dimension_set_since_prior_landmark；
first_observed_failure_time_P_as_of_landmark；
first_observed_failure_time_C_as_of_landmark；
first_observed_failure_time_T_as_of_landmark；
first_observed_failure_time_V_as_of_landmark；
failed_dimension_count_as_of_landmark；
strict_core_member；
```

#### 状态几何特征

```text

qualified duration；
zone trading span；
confirmed density；
component_count；
bridge_count；
收敛期间的价格宽度和变化速度；
```

#### T0/T1/T2 价格路径特征

```text
相对状态期中枢、上沿和下沿的位置；
严格过去尺度标准化的位移；
```

#### 市场环境特征

```text

个股相对行业/指数强弱；
市场广度；
可审计的市场波动与流动性环境。
```

所有特征必须有 `feature_available_time`。不得使用 landmark 之后的信息，也不得用未来 H 日统计量回填 T0/T1/T2 特征。

“维度失效”字段必须满足：

```text
exit_trigger_dimension_set_at_T0
    只包含 T0 当天相对上一有效 confirmed 状态已经观察到的失效维度；

failed_dimension_set_as_of_landmark
    只包含截至当前 landmark_time 已经观察到的失效维度；

first_observed_failure_time_*_as_of_landmark
    若非空，必须 <= landmark_time；

landmark 之后才发生的维度失效
    不得回填到当前或更早的特征快照。
```

机器断言至少包括：

```text
feature_available_time <= landmark_time；
first_observed_failure_time_* <= landmark_time；
newly_failed_dimension_set_since_prior_landmark
    ⊆ failed_dimension_set_as_of_landmark；
T0 快照不得出现 first_observed_failure_time > T0；
```

## 13. 样本隔离与标签治理

至少划分：

```text
design sample：估计标准化、对照匹配、经验边界和标签技术规则；
validation sample：验证边界校准、标签稳定性和技术状态比例；
out-of-sample / forward sample：只使用冻结边界和标签引擎分配结果。
```

即使标签生成包含无监督或经验分布估计，也不得在全部历史和样本外时期上共同学习边界、缩放参数或转折门槛。

同一 `event_id` 的全部 exit attempts 必须进入同一分区。分区键不得只使用 `exit_attempt_id` 随机切分。最长标签窗口 H30 与最大 Landmark 延迟 T2 必须进入 purge / embargo 设计，避免相邻分区共享未来结果区间。

禁止：

```text
根据状态事件的未来收益选择边界误差水平；
根据 R4 模型准确率修改五类路径定义；
根据回测收益改变 H20 主窗口；
删除 NO_CLEAR_PATH 以提高模型准确率；
在验证或样本外数据上重新估计边界；
只保留成功向上或向下的事件。
```

```text
误差容忍政策；

边界版本和设计样本；
```

## 15. 验收条件

R3 最低验收门槛如下：

1. **输入门槛**：只读取 R2 final freeze manifest 登记的 frozen versions，hash、schema、lineage 闭合。
2. **T0 门槛**：每一次合法 T0 均来自 qualified event 内 `confirmed_state=true → natural_state_exit` 的真实转换；同一 event 的后续合法 attempts 未被删除或降级，质量中断和普通 raw=false 未混入。
3. **g 边界门槛**：T0 与 event finalization 分离；后续 same-zone merge 不删除 T0；g-resolution 字段完整。
4. **对照门槛**：control 风险集、匹配、标准化和样本支持可审计；不得使用未来信息。
5. **边界门槛**：上下边界来自 design sample 的普通路径分布，随 horizon 响应合理，并在验证样本中完成校准检查。
6. **标签门槛**：五类规则互斥、可重放；RECONVERGENCE、同日顺序冲突和复杂多转折处理明确。
7. **窗口门槛**：H20 是唯一 primary path class，H5/H10/H30 角色清晰，transition matrices 完整。
8. **技术状态门槛**：NO_CLEAR、AMBIGUOUS、COMPLEX、CENSOR 和 ABNORMAL 不被静默删除。
9. **Landmark 门槛**：T0/T1/T2 特征时点一致；维度失效字段严格 as-of；`landmark_scoring_eligible`、`target_observable_H` 和训练/验证 eligibility 分层正确；右删失或 NULL 标签未进入监督目标集。
10. **结果审阅门槛**：正式 artifacts 已被实际读取、独立复算、异常扫描并提交 result analysis；不能只接受 validator counts。
11. **结论边界门槛**：R3 只冻结标签语言和数据接口，不宣称预测或交易优势。

## 18. 回退与变更控制

```text
PCVT 指标、W/q/K 或状态逻辑变化
→ 最低退回 R0，并重新经过 R1/R2。

confirmed interval、d、g、event ID 或 R2 state-machine 变化
→ 最低退回 R2。

T0、exit-attempt 主分析单位、attempt 枚举规则、control、标准化、经验边界、五类标签、技术状态、H20 primary、landmark 字段或 target-observability 规则变化
→ 退回对应 R3 task，并重跑其后全部 R3 tasks。

R4 模型、概率阈值、拒判规则或解释方法变化
→ 留在 R4，不得修改同一版本 R3 标签。

交易时点、止盈止损、成本、持有和组合约束变化
→ R5。
```

若未来希望把 `NO_CLEAR_PATH` 升级为第六类经济主路径，或新增“三段式”路径类别，必须创建新的 path ontology version，不得覆盖 v0.3 的历史标签。

## 19. R3 到 R4 的交接原则

R3 交给 R4 的不是“有效突破交易信号”，而是：

```text
全量合法 T0 exit attempts 及其 event/component/ordinal 上下文；
普通路径经验边界；
五类主路径和技术状态；
H5/H10/H20/H30 标签；
T0/T1/T2 时点一致特征；
样本切分、删失和 limitation；
```

R4 必须遵守：

```text
只读取 R3 final label manifest 登记的 artifacts；
不得重算或改写未来路径标签；
不得删除 NO_CLEAR、AMBIGUOUS 或失败样本以提高准确率；
不得把 T1/T2 已观察信息当作判断后的未来结果；
必须分别报告 T0/T1/T2 的判断覆盖率、概率校准和高置信度错误；
必须区分完整 T0 后路径与实际 landmark 后剩余路径；
不得用 R4 结果反向修改 R2/R3 定义。
```

R3 的最终含义是：

> R2 冻结“什么是收敛事件”；R3 对该事件中每一次从已确认收敛状态发生的合法自然退出生成独立 T0，利用普通时期的经验路径分布定义什么叫超出噪声的方向移动，并按重新收敛与上下边界穿越顺序，把每次 attempt 的未来路径冻结为五类主标签和技术状态；R4 再研究在 T0、T1、T2 当时可得的信息下，能否可靠估计这些路径的条件概率。
