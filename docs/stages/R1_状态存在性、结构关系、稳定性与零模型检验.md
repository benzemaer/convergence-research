# R1：状态存在性、结构关系、稳定性与零模型检验

> 文档类别：Structural Validation Specification
> 所属阶段：R1
> 状态：草案 v0.3（研究路线整理版）
> 前置依赖：R0 候选状态日表、确认区间表与 manifest（冻结数据版本下生成）
> 后续阶段：R2 状态冻结
> 生效边界：R1 只做结构验证，不定义未来标签，不使用未来收益、未来波动扩张、未来突破方向、回测收益或交易结果选择指标及参数；R1 不冻结最终状态版本，R2 才冻结。

本版保留 R1-T01 至 R1-T04 的既有设计，重新拆分 R1-T05 至 R1-T08 的研究职责，并增加层级 `q` 解耦候选的触发式复验接口。R1-T03 的 27 组 `W/q/K` 全量轻量结构扫描已完成；具体运行结果与验收证据由对应 task artifact 保存，本总纲只保留阶段职责、证据边界与后续路线。

---

## 1. R1 阶段目标

R1 的目标是独立验证 R0 产出的候选状态是否构成可辨识的结构现象，而不是指标噪声、P 层低波动状态的同义重复、滚动指标的天然同步，或数据质量问题造成的机械共现。

R1 需要回答四个问题：

1. **状态是否存在**：`S_PCT` 与 `S_PCVT` 是否非空、非退化，并具有足够的状态日、股票数、确认区间与年份覆盖。
2. **状态是否有结构**：同层指标是否方向一致但保留补充信息；C/T/V 是否在 P、PC、PCT 条件风险集内提供额外结构，而不是机械缩窄样本。
3. **结构是否稳定**：结构结论是否在合理 `W/q/K` 配置和年份切片下保持方向一致，而不是只在一个孤立配置或单一年份成立。
4. **同步是否超过零模型**：观察到的 `S_PCT` / `S_PCVT` 共现是否超过“保留各层自身边际分布、时间自相关和持续性，但破坏跨层同期对齐”的随机错位结果。

R1 的结论只能表述为结构结论，例如“该状态线在当前定义下具备进入 R2 冻结讨论的结构证据”。R1 不得表述为“该状态未来会上涨”“该状态有交易优势”“该参数组合收益最好”。

## 2. 阶段定位与依赖边界

研究主线如下：

```text
D0 → D1 → D2 → D3
               ↓
              R0 → R1 → R2 → R3 → R4 → R5 → R6
```

R1 只能读取 R0 授权交接的候选状态产物，包括 candidate daily state、confirmed interval、manifest、config hash、schema、quality flags 与 `confirmation_time`。R1 不绕过 R0 读取 raw / MarketDB，也不在 R1 内重新计算 strict-past percentile。

R1 是 **Structural Validation Gate**。R1 的输出是 R2 decision matrix，而不是最终 frozen state。R2 才能冻结最终 `W/q/K`、状态线版本、确认规则、区间规则和后续事件接口。

若 R2 后续提出新的 layer-specific `q` 候选，职责链必须是：

```text
R2 诊断共享 q 是否需要解耦并提出候选 q_vector
  ↓
R0 按 strict-past 规则物化新的候选状态产物
  ↓
R1 对新候选执行触发式结构复验
  ↓
R2 决定是否冻结
```

R2 不得将未经过 R0 物化和 R1 结构复验的新 `q_vector` 直接冻结。

## 3. 已确认的 R1 研究取舍

### 3.1 `W/q/K` 的用途

R0 主网格为：

```text
W ∈ {120, 250, 500}
q ∈ {0.10, 0.20, 0.30}
K ∈ {2, 3, 5}
```

共 27 组 strict 配置。

这 27 组不是参数搜索池，不用于按未来表现选择参数，也不用于跑完后挑最小 p-value 或最大 JointLift 的组合。它们在 R1 中承担三类作用：

1. **全量轻量结构扫描**：27 组全部进入 early light profile，用于画清楚状态空间、覆盖率、碎片率、年份覆盖、确认区间和基础条件保留率。
2. **预注册配置正式验证**：正式结构检验与零模型优先用于 baseline 与 state-line challenger，而不是一开始将 27 组全部作为 primary hypothesis test。
3. **触发式全量零模型**：只有 baseline / challenger 结论冲突，或 R2 明确需要 family-level 选择证据时，才对 27 组补跑全量零模型；即便补跑，也必须使用 family-level correction 或 max-statistic null，不得按单组最优统计量直接选参数。

`q∈{0.10,0.20,0.30}` 在 R1 中是 coarse sensitivity knob，用于检验结构对严格、中等、宽松低尾口径是否稳健，不用于估计小数点级“最优 q”。

### 3.2 baseline 的地位

`W=250,q=0.20,K=3` 是 **reference baseline**，不是默认最优参数。它提供中性参照：一年左右历史窗口、中等低尾阈值、中等确认长度。

R1 评价 baseline 时，判断的是它是否具备结构证据，而不是它是否未来表现最好。如果 baseline 通过，且 challenger 没有明显结构优势，baseline 可作为 R2 优先候选。如果 baseline 通过但某条状态线的 challenger 更符合构念语义且证据稳定，R1 可将 challenger 一并交给 R2 review。如果 baseline fail，而 challenger 在同一规则下 pass，R1 可将 challenger 标为 `review_candidate`，但仍不在 R1 冻结。

### 3.3 `S_PCT` 与 `S_PCVT` 分线研究

R1 必须分别研究两条状态线：

```text
S_PCT  = P ∧ C ∧ T
S_PCVT = P ∧ C ∧ T ∧ V
```

`S_PCT` 表示不要求参与度枯竭的结构收敛状态。它可能更敏感，可能更适合较短历史窗口，例如 `W=120`。这是 R1 需要检验的假设，而不是预设结论。

`S_PCVT` 表示在同参数 PCT parent 基础上叠加 V 的枯竭型结构收敛状态。它可能更需要中长历史窗口，例如 `W=250`，因为 V 层如果历史窗口太短，可能更容易被短期成交 regime 干扰。这同样是 R1 需要检验的假设。

因此，R1 允许最终向 R2 提交 state-line-specific 候选，例如：

```text
S_PCT_fast_candidate          # 例如 W=120,q=0.20,K=3，若证据支持
S_PCVT_depletion_candidate    # 例如 W=250,q=0.20,K=3，若证据支持
```

R1 不强迫 `S_PCT` 与 `S_PCVT` 共用同一组 `W/q/K`。但每一个 `S_PCVT` 候选都必须使用同参数的 `S_PCT_parent` 检查其父子关系和 `V_given_PCT` 增量；不能用 `S_PCT_fast@W120` 作为 `S_PCVT_depletion@W250` 的逐行父状态。

### 3.4 raw 与 confirmed 双主线

raw state 用于观察定义本体、单指标边界、层内互补性和 raw fragment。confirmed state 用于实时可得性、K 影响、区间结构、正式零模型主证据和 R2 交接。

R1 的判断规则是：

```text
raw       → 诊断定义本体
confirmed → 决定是否具备 R2 freeze_candidate / review_candidate 资格
```

R2 decision matrix 的主状态应以 confirmed state 与 `confirmation_time` 为准。raw 结果只能作为诊断、解释或 warning 来源，不能单独使某配置进入 `freeze_candidate`。

### 3.5 R1 证据层级

R1 的四类结构证据具有不同职责：

```text
层内互补性：诊断同层构件是否同义、冲突或互补；主要产生解释与 warning。
层间同期增量：检验 C/T/V 在 P、PC、PCT 风险集中的留存、富集和额外约束信息。
固定滞后关系：描述 P onset 后各层的时序关联；属于 secondary evidence，不要求必须显著。
同步性与嵌套增量零模型：R1 的正式统计门禁，检验完整状态同步及新增层增量是否超过随机错位。
```

一个 contemporaneous state 不需要预设固定的 `P→C→T→V` 时间顺序才能成立。因此，固定滞后关系没有显著结果时，不得仅据此否定 `S_PCT` 或 `S_PCVT`。

### 3.6 零模型主口径

正式零模型分为 global synchronization null 和 nested increment null。

#### 3.6.1 Global synchronization null

```text
S_PCT：P 保持不变，C/T 在 security_id × year × continuous trading segment 内分别 circular shift。
S_PCVT：P 保持不变，C/T/V 在相同块内分别 circular shift。
```

它保留各层边际分布、块内自相关和持续结构，破坏跨层 contemporaneous alignment，用于检验完整 `S_PCT` / `S_PCVT` 同步是否超过机械重合。

#### 3.6.2 Nested increment null

```text
C 增量：P 保持不变，仅 circular shift C。
T 增量：P/C 保持不变，仅 circular shift T。
V 增量：P/C/T 保持不变，仅 circular shift V。
```

其中 `PCT-fixed V-shift` 是判断 V 是否在 `P=C=T=1` 条件风险集内提供额外结构信息的正式检验。Global `S_PCVT` null 通过，不能替代 V 的 nested increment 检验。

#### 3.6.3 Monte Carlo 设定

```text
默认正式 N_perm = 2000；
p = (n_extreme + 1) / (N_perm + 1)；
默认最小可报告 p = 1/2001 ≈ 0.0005。
```

主证据是 JointLift、JointExcess 与有限样本经验 p 值。z-score 只作描述，不作为主结论。若结果接近 pass / warning / fail 边界，或启动 27-grid family-level null，则将 `N_perm` 升级为 10000，并沿用已冻结的 seed、block 与 offset plan。

`CT-bundle shift`、`CTV-bundle shift`、无锚平移、分块置换或块长 B 变体可以作为 sidecar，用于检查零模型设定是否影响结论，但不替代主口径。

### 3.7 年份稳定性作为 R1 基础稳定性

当前基础数据库稳定可用的跨样本字段按交易日派生为年份。因此 R1 v0.3 的基础稳定性只强制检查 **年份稳定性**。

市场状态、流动性层、股票年龄、行业、交易约束和质量归因不进入 R1 基础门槛，除非后续数据层提供了版本化、时点一致、可审计字段。没有 as-of 字段时，R1 不应硬造这些分层，也不应把它们写成阻断性验收条件。

### 3.8 层级 `q` 解耦的阶段边界

共享 `q` 是 R0/R1 主线的简洁 baseline。细粒度或 layer-specific `q` 扫描不属于当前 R1 必做路线。

只有出现以下触发情形时，R2 才应提出层级 `q` 解耦候选：

```text
某层在 q=0.10/0.20/0.30 下持续成为主要样本瓶颈；
某层的同期增量在三档 q 下方向不稳定；
global synchronization null 通过，但对应 nested increment null 不通过；
S_PCT 与 S_PCVT 对共享 q 呈现明显不同的结构需求；
R2 无法在当前 coarse grid 中形成语义合理、非孤立的冻结区域。
```

R2 输出候选 `q_vector` 与触发理由，R0 负责物化新候选，R1-T14 只做结构复验。R1-T14 不自行遍历连续阈值，不按最大 JointLift、最小 p-value 或未来标签选择 q。

## 4. R1 primary hypothesis tests

### 4.1 完整状态同步检验

对某条状态线和某个预注册配置，检验：

> 观察到的多维状态同步是否超过“保持各层自身分布和时间结构、但破坏跨层同期对齐”的 global null。

对 `S_PCT`，主检验回答：

```text
P、C、T 的同步共现是否超过随机错位后的自然共现。
```

对 `S_PCVT`，主检验回答：

```text
P、C、T、V 的同步共现是否超过随机错位后的自然共现。
```

### 4.2 嵌套增量检验

完整状态同步成立后，还要分别检验：

```text
C 是否在 P 风险集内提供超过随机错位的增量；
T 是否在 PC 风险集内提供超过随机错位的增量；
V 是否在 PCT 风险集内提供超过随机错位的增量。
```

`S_PCT` 的结构资格依赖 C/T 增量；`S_PCVT` 的结构资格依赖同参数 PCT parent 的结构基础、V 的 nested increment，以及完整 PCVT global synchronization evidence。

### 4.3 固定滞后检验的地位

P onset 固定滞后分析回答的是：

> P 首次进入后，C/T/V/PCT/PCVT 在预声明滞后处的出现概率是否高于相应基准或经验零分布。

它用于描述状态可能的形成顺序，不证明因果，也不作为完整状态成立的必要条件。

### 4.4 禁止解释

R1 primary hypothesis tests 不得被解释为：

```text
W=250,q=0.20,K=3 是最优参数；
S_PCVT 之后会上涨；
W=120 比 W=250 更能带来交易收益；
27 组里 p-value 最小的一组就是最终定义；
P 导致 C/T/V；
当前零模型已经排除所有共同市场驱动或样本选择机制。
```

参数比较属于 configuration comparison / structural model selection。R1 只能基于预注册结构规则向 R2 推荐 `freeze_candidate` 或 `review_candidate`。

## 5. R1 工作路线

### R1-0：冻结验证协议

R1 首先冻结 `R1_config_manifest`，包括：

```text
reference_config = W250_q20_K3
all_27_configs_light_profile = true
state_lines = S_PCT, S_PCVT
raw_confirmed_mode = dual_line
r2_decision_basis = confirmed_state
primary_global_null = P_fixed_independent_layer_circular_shift
nested_increment_nulls = P_fixed_C_shift, PC_fixed_T_shift, PCT_fixed_V_shift
N_perm = 2000
boundary_or_family_N_perm = 10000
lag_set = {1,3,5,10,20}
year_stability_required = true
future_labels_forbidden = true
```

R1-T01 之后，除非发现 R0 产物错误或 manifest 不一致，不得因结果好坏修改主检验规则。

### R1-1：R0 产物接收与时点一致性复检

R1 只读取 R0 授权交接产物，复检：

```text
candidate daily state 是否来自冻结 R0 run
confirmed interval 是否有 confirmation_time
manifest / schema / config hash 是否完整
historical percentile 是否严格使用 τ < t
confirmation_time 是否未回填为 raw_start_date
unknown / blocked 是否未被写入否定状态
forbidden columns 是否不存在
```

若此关失败，配置或状态线标为 `blocked_return_to_R0`，不能继续解释结构结果。

### R1-2：27 组 `W/q/K` 全量轻量结构扫描

27 组全部输出 light profile。每组至少报告：

```text
raw coverage
confirmed coverage
state days
unique securities
confirmed intervals
nonzero years
year coverage
unknown / blocked ratio
fragment rate
average duration
S_PCT coverage
S_PCVT coverage
C_given_P retention
T_given_PC retention
V_given_PCT retention
relative_to_baseline
```

这一步用于识别参数空间形态，不直接选择参数。

当前状态：**R1-T03 已完成**。具体 run_id、修复验收、逐配置结果和 acceptance evidence 保存在 R1-T03 task artifact 中；后续任务直接消费其正式产物，不在总纲中复制结果表。

### R1-3：`S_PCT` 与 `S_PCVT` 分线状态画像

`S_PCT` 的 reference / challenger：

```text
S_PCT @ W=250,q=0.20,K=3     # reference baseline
S_PCT @ W=120,q=0.20,K=3     # fast structural convergence challenger
S_PCT @ W=120,q=0.20,K=2     # optional K sidecar
```

要验证的问题是：`W=120` 是否更敏感地捕捉 PCT 结构收敛，同时不显著恶化碎片率、年份集中度和零模型证据。

`S_PCVT` 的 reference / challenger：

```text
S_PCVT @ W=250,q=0.20,K=3    # reference baseline / depletion candidate
S_PCVT @ W=120,q=0.20,K=3    # short-window counterfactual
S_PCVT @ W=500,q=0.20,K=3    # optional long-window sidecar
S_PCVT @ W=250,q=0.20,K=5    # optional stricter-confirmation sidecar
```

要验证的问题是：中长窗口是否更适合 V 的参与度枯竭构念，是否保留更稳定的 `V_given_PCT` 增量和更连贯的 confirmed interval。

### R1-4：单指标诊断与层内互补性

每项指标输出：

```text
mean
standard deviation
median
1%, 5%, 10%, 20%, 30%, 50%, 90%, 95%, 99% quantiles
missing ratio
extreme value ratio
historical percentile distribution
consecutive low-percentile days
single-day fragment ratio
```

层内关系以两项指标共同 eligible 的样本为 denominator，优先在方向统一后的连续 score 或 strict-past historical percentile score 上计算 Spearman；unknown / blocked 不得当作未命中。

每层至少输出完整阈值命中结构：

```text
both_hit
indicator_a_only
indicator_b_only
neither
A_given_B
B_given_A
Jaccard
joint-state duration
```

目标不是相关性越高越好，也不是相关性越低越互补，而是确认同层指标方向一致、共同命中非退化，同时保留非平凡的独立约束信息。输出可标记：

```text
complementary_structure
redundancy_warning
construct_conflict_warning
insufficient_eligible_sample
```

如果同层指标近似同义或存在冲突，R1 只标注并交给 R2，不在 R1 自行删除或改写指标。

### R1-5：层间同期留存、关联 Lift 与嵌套增量

R1 分别报告留存率、相对富集和绝对增量。`C_given_P`、`T_given_PC`、`V_given_PCT` 是 retention，不应单独称为 Lift。

留存率：

```text
C_given_P   = P(C=1 | P=1)
T_given_PC  = P(T=1 | P=C=1)
V_given_PCT = P(V=1 | P=C=T=1)
```

关联 Lift 与绝对增量：

```text
Lift_C  = P(C=1 | P=1)     / P(C=1)
Lift_T  = P(T=1 | PC=1)    / P(T=1)
Lift_V  = P(V=1 | PCT=1)   / P(V=1)

Delta_C = P(C=1 | P=1)     - P(C=1)
Delta_T = P(T=1 | PC=1)    - P(T=1)
Delta_V = P(V=1 | PCT=1)   - P(V=1)
```

所有条件关系必须使用同一 `candidate_config_id` 的上游 parent，并明确 common eligible denominator。raw 结果用于构念诊断，confirmed 结果用于 R2 交接判断。

这一步回答“新增一层是否只是机械缩窄样本，还是在上游风险集中出现了非平凡富集”。正式的统计门禁由 R1-T08 的 nested increment null 给出；T06 不按最大 Lift 选择参数。

### R1-6：P 首入锚定的固定滞后结构关系

P onset 定义为同一股票上一 eligible trading day 未处于 P、当前日进入 P。上一日为 unknown / blocked 时，不认定为 onset；一个连续 P run 只产生一个 onset。

正式风险集使用所有 eligible P onsets，不得只保留后来进入 PCT / PCVT 的 onset。

固定滞后集合为：

```text
k ∈ {1,3,5,10,20}
```

不允许密集 lag-grid 搜索并挑选效果最好的滞后。

目标层包括：

```text
P_to_C
P_to_T
P_to_V
P_to_PCT
P_to_PCVT
```

每个 transition path × lag 至少输出：

```text
anchor_event_count
eligible_target_count
observed_probability
baseline_or_null_probability
absolute_lift
relative_lift
null_interval
empirical_p_or_descriptive_status
```

`P_to_PCT` 与 `P_to_PCVT` 会同时受到 P 自身持续性的影响，因此应同步报告 `P survival at k`，避免把 P 持续机械解释成后续层形成。

固定滞后结果属于 secondary structural evidence。结果不显著不等同于状态结构失败；结果显著也不得表述为因果关系。

### R1-7：同步性与嵌套增量零模型检验

优先对以下预注册配置执行正式零模型：

```text
S_PCT  @ W=250,q=0.20,K=3
S_PCT  @ W=120,q=0.20,K=3
S_PCVT @ W=250,q=0.20,K=3
S_PCVT @ W=120,q=0.20,K=3
```

触发式 sidecar：

```text
S_PCT  @ W=120,q=0.20,K=2
S_PCVT @ W=500,q=0.20,K=3
S_PCVT @ W=250,q=0.20,K=5
```

正式实现至少包含：

```text
global S_PCT null：P fixed，C/T independent circular shift
global S_PCVT null：P fixed，C/T/V independent circular shift
C nested null：P fixed，C shift
T nested null：PC fixed，T shift
V nested null：PCT fixed，V shift
```

置换作用于构成联合状态的单层 daily flags，并按与 observed 完全相同的 AND、K 与 confirmation pipeline 重建 null state；不得直接平移最终联合状态来替代层级同步检验。

零模型输出至少包括：

```text
observed value
null mean
null median
null interval
JointLift
JointExcess
empirical p
N_perm
n_extreme
seed policy
failed simulation count
observed duration / null duration
observed fragment rate / null fragment rate
```

coverage 与 duration 默认使用 upper-tail；fragment rate 默认使用 lower-tail；窗口配对差异默认使用 two-sided，除非 R1-T01 预先声明方向。

主结论只能表述为：观察到的结构不能由当前预设的随机错位零模型充分解释。不得表述为已排除全部共同市场驱动、自相关机制或样本选择机制。

### 触发式分支：R1-T14 层级 `q` 解耦候选结构复验

R1-T14 在逻辑上位于 R1-T08 之后，只在 R2 提出正式 `q_decoupling_request` 且 R0 已物化新候选后启用。它不是当前 R1 主路线的前置条件。

对新 `q_vector`，至少复验受影响的：

```text
R1-T05 阈值共现与单项命中结构
R1-T06 同期留存、Lift 与 Delta
R1-T08 global / nested null
R1-T09 年份稳定性
R1-T10 R2 decision matrix 条目
```

R1-T14 不自行做连续阈值优化，不将最小 p-value 或最大 JointLift 作为 q 选择器，也不得引入未来标签。

### R1-8：年份稳定性检查

R1 v0.3 的基础稳定性只按年份检查。年份由交易日派生，例如 `year = YEAR(trade_date)`。

每个 `state_line × candidate_config_id × year` 至少输出：

```text
eligible trading days
raw state days
confirmed state days
confirmed coverage
unique securities
confirmed intervals
average duration
fragment rate
unknown / blocked ratio
C_given_P
T_given_PC
V_given_PCT
```

年份稳定性不是要求每年频率相等。R1 只检查方向和非集中性：

```text
状态不能只集中在单一年份；
核心结构关系不能只在单一年份成立；
某一年为 0 或异常高时必须解释为样本不足、数据问题、制度差异或真实结构差异；
如果年份覆盖不足，标为 review_candidate 或 insufficient_year_coverage，而不是强行通过。
```

市场状态、流动性层、股票年龄、行业等分层不作为 R1 v0.3 基础要求。若未来数据层提供 as-of 字段，可新增 optional stability sidecar。

### R1-9：R2 交接矩阵

R1 最终输出 `R2_decision_matrix`。进入 formal evaluation 的每条状态线和候选配置给出一个交接状态：

```text
freeze_candidate
review_candidate
do_not_freeze
blocked_return_to_R0
```

这些状态的含义见 §6。

## 6. R2 交接状态定义

R1 不冻结最终参数，只给 R2 交接状态。交接状态是对完成 formal evaluation 的 `state_line × candidate_config_id` 的判定，例如：

```text
S_PCT  × W120_q20_K3
S_PCT  × W250_q20_K3
S_PCVT × W250_q20_K3
S_PCVT × W120_q20_K3
```

### 6.1 `freeze_candidate`

`freeze_candidate` 表示：该状态线与配置已经通过 R1 的结构门槛，具备进入 R2 冻结讨论的资格。

它必须至少满足：

```text
R0 lineage / manifest / schema / config hash 通过；
无前视复检通过；
confirmed state 非退化；
状态日、股票数、确认区间和年份覆盖达到 R1-T01 预注册地板；
unknown / blocked 未主导结果；
层内指标不存在使构念失效的完全同义或方向冲突；
层间同期留存、Lift 与关键 nested increment 方向合理；
完整状态 global null 达到预注册门槛；
对应新增层的 nested increment null 达到预注册门槛；
年份稳定性未显示单一年份主导。
```

`S_PCT` 候选需要 C/T 增量与 global PCT synchronization evidence；`S_PCVT` 候选还需要同参数 PCT parent 的结构基础、V nested increment 和 global PCVT synchronization evidence。

固定滞后关系不要求必须显著，不单独决定 `freeze_candidate`。

注意：`freeze_candidate` 不是 frozen。它只是“R2 可以审议冻结”。R2 仍可因为版本治理、语义冲突、参数简洁性或与其他状态线不一致而不冻结它。

### 6.2 `review_candidate`

`review_candidate` 表示：该状态线与配置有一定结构证据，但存在 material warning，需要 R2 或补充 sidecar 审查。

典型情形包括：

```text
baseline pass，但 challenger 在某条状态线明显更符合构念；
global null 通过，但某个 nested increment 边界或偏弱；
零模型通过，但年份集中度偏高；
confirmed coverage 偏低但未低到直接 fail；
K 选择敏感，K=2/3/5 的结论差异较大；
W=120 与 W=250 对同一状态线给出冲突结论；
固定滞后关系与同期结构不一致，需要解释但不构成直接 fail；
某些年份为 0 或异常高，需要 R2 判断是否接受；
共享 q 可能形成层级瓶颈，需要触发 R2 q 解耦诊断。
```

`review_candidate` 不能在 R1 自动升级为 frozen。若 R2 接受 warning，需要在 R2 decision record 中写明接受理由；若不接受，则退为 `do_not_freeze` 或要求补跑 sidecar。

### 6.3 `do_not_freeze`

`do_not_freeze` 表示：该状态线与配置在当前 R1 协议下不具备进入 R2 冻结讨论的资格。

典型情形包括：

```text
状态几乎全 0 或全 1；
状态日、股票数、区间数或年份覆盖不足；
unknown / blocked 主导；
层内指标完全同义或方向冲突到使构念失效；
C/T/V 在对应上游风险集内没有结构增量；
完整状态 global JointLift ≤ 1 或 empirical p 未达到预注册门槛；
对应 nested increment null 未通过且不存在可解释的边界证据；
结构只在单一年份成立且无法解释。
```

`do_not_freeze` 不是说构念永远无效，只表示当前 R0 产物与当前 R1 协议下不能进入 R2 冻结。

### 6.4 `blocked_return_to_R0`

`blocked_return_to_R0` 表示：R1 无法合法解释该配置的结构结果，原因不是“状态结构不成立”，而是输入产物或时点一致性前提不成立。

典型情形包括：

```text
R0 manifest 缺失或 hash 不一致；
schema 不匹配；
关键字段缺失；
confirmation_time 回填；
historical percentile 使用了 τ ≥ t 的数据；
unknown / blocked 被错误写成 false state；
候选状态表与确认区间表无法对齐；
R0 run_id / data_version / config_id 无法追溯。
```

这种状态必须退回 R0 修复。它不能被当作 R1 fail，也不能在 R1 内临时修补后继续使用。

## 7. R1 验收条件

R1 的验收不是“找到显著结果”，而是证明候选状态经得起预先声明的结构门槛。

最低门槛如下：

1. **输入门槛**：R0 lineage、manifest、schema、config hash 与无前视复检通过。
2. **存在性门槛**：被验证状态非空、非退化，并具有足够状态日、股票数、确认区间与年份覆盖。
3. **层内诊断门槛**：同层指标不存在使构念失效的完全同义或方向冲突；一般冗余只产生 warning，不由 R1 擅自删指标。
4. **层间增量门槛**：`C_given_P`、`T_given_PC`、`V_given_PCT` 的留存、Lift 与 Delta 方向合理；对应 nested increment null 提供正式证据。
5. **完整同步门槛**：预注册配置的 global JointLift、JointExcess 和经验 p 值达到 R1-T01 设定门槛；z-score 只作描述。
6. **固定滞后证据**：完成预声明滞后计算并报告风险集与不确定性；不要求所有路径或滞后显著，不作为单独否决门槛。
7. **年份稳定性门槛**：核心结论不能只由单一年份主导；若年份覆盖不足或年份差异过大，必须降级为 `review_candidate` 或 `do_not_freeze`。
8. **解释边界门槛**：报告不得使用未来预测、因果、最优参数或排除全部共同驱动等越界表述。

## 10. 必须 task 与可选 task 的边界

R1-T01 至 R1-T10 是进入 R2 的最低必要路线。它们覆盖验证协议、R0 产物合法性、27-grid early profile、状态线分线画像、单指标与层内互补性、层间同期增量、固定滞后、正式 global / nested null、年份稳定性和 R2 交接矩阵。

R1-T11 至 R1-T14 是触发式 sidecar。它们只在以下情形启用：

```text
baseline 与 challenger 结论冲突；
R2 要求 family-level 参数选择证据；
主零模型设定成为结论敏感点；
R0 已有替代指标口径且 baseline 触发 warning；
R2 诊断共享 q 存在层级瓶颈并提出新的 q_vector。
```

R1-T14 的逻辑依赖位于 R1-T08 之后。若它在初次 R2 交接前触发，R1-T09/T10 必须消费其结果；若它由 R2 在初次交接后触发，则作为 R1 extension run 重新生成受影响的年份稳定性与 decision matrix 条目。

所有可选 task 均不得引入未来收益、未来波动、未来方向、未来路径、回测收益或交易结果。

## 11. R1 到 R2 的交接原则

R1 交给 R2 的不是“最终答案”，而是结构证据矩阵。R2 需要基于 R1 的 `R2_decision_matrix` 决定是否冻结：

```text
某条状态线；
某组 W/q/K 或经过复验的 q_vector；
某个确认规则；
某种区间规则；
某个状态版本名称。
```

R1 可以推荐多个候选并存。例如，若证据支持，R1 可以同时交接：

```text
S_PCT_fast_candidate @ W=120,q=0.20,K=3
S_PCVT_depletion_candidate @ W=250,q=0.20,K=3
```

这不构成冲突，因为 `S_PCT` 与 `S_PCVT` 对应不同构念和场景。对于 `S_PCVT_depletion`，R1 仍需保留同参数 `S_PCT_parent_depletion@W250` 作为嵌套增量和父子关系的审计对象。

层级 `q` 解耦候选的交接必须同时附带：

```text
R2 q_decoupling_request；
R0 新候选 manifest 与 lineage；
R1-T14 revalidation evidence；
新的 R2 decision matrix 条目。
```

最终是否冻结为一个或多个状态版本，由 R2 决定。R2 可以拒绝所有候选；R1/R2 不承担必须冻结至少一个版本的义务。
