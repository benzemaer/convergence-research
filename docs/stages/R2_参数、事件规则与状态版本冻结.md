# R2：参数、事件规则与状态版本冻结

> 文档类别：State Freeze Specification
> 所属阶段：R2
> 状态：草案 v0.1（基于 R1-T10 正式交接矩阵）
> 前置依赖：R1-T10 final gate、12 行 `R2_decision_matrix`、R0-T10 / R0-T15 正式状态产物与完整 lineage
> 后续阶段：R3 释放、风险集与标签设计
> 生效边界：R2 只冻结状态参数、事件区间规则、状态版本和 canonical 产物；不重新执行 R1 的结构发现与零模型，不使用未来收益、未来波动、未来方向、未来路径、release label、precision / recall、回测结果或交易结果选择参数。

---

## 1. R2 阶段目标

R2 的目标是消费 R1 已完成的结构验证证据，将具备结构资格的候选状态收敛为可审计、可复现、可由 R3 唯一消费的冻结状态版本和事件接口。

R2 需要回答四个问题：

1. **参数候选如何收敛**：R1-T10 交接的 shared-q 与 layer-specific q-vector 候选中，哪些进入正式事件建模，哪些只保留为 strict core、sensitivity 或被拒绝。
2. **事件规则如何定义**：在 `K=3` confirmed state 基础上，如何使用 `d` 和 `g` 将日度状态转换为事件区间，并保证所有可用时点无前视。
3. **哪些状态版本最终冻结**：对四条平等候选路线执行 hard gate、Pareto / complexity 比较和人工决策，允许冻结 0–4 个正式版本。
4. **如何形成 canonical 产物**：只物化最终选中的版本，完成回放、异常扫描、独立审阅和最终 freeze manifest，再向 R3 提供唯一正式入口。

R2 的结论只能表述为状态和事件接口的冻结结论，例如：

```text
某个 state_line × W × q-vector × K × d × g 已通过 R2 门禁，
并被冻结为可供 R3 使用的状态版本。
```

R2 不得表述为：

```text
该状态未来会上涨；
该窗口对应确定的交易周期；
该版本具有交易优势；
该参数是数学或市场意义上的全局最优；
某个 release / direction / return label 已经成立。
```

## 3. 已确认的 R2 研究取舍

### 3.1 四条 q-vector 路线平等进入事件建模

R2-T01 的 primary shortlist 固定为四条路线：

```text
S_PCT  × W120 × K3 × qP=.20,qC=.20,qT=.25,qV=.20
S_PCT  × W250 × K3 × qP=.20,qC=.20,qT=.25,qV=.20
S_PCVT × W120 × K3 × qP=.20,qC=.20,qT=.20,qV=.30
S_PCVT × W250 × K3 × qP=.20,qC=.20,qT=.20,qV=.30
```

四条路线以平等身份进入 R2-T03 的 `d×g` 扫描和 R2-T04 的最终处置。R2 不预先指定：

```text
S_PCT 必须选择 W120；
S_PCVT 必须选择 W250；
W120 或 W250 是另一窗口的优胜者或等价表达。
```

现有 R1 证据只证明 W120 与 W250 均有结构资格，并且状态身份存在明显差异；没有证明某个窗口天然属于某条状态线。

### 3.2 q-vector 是 primary candidate，shared-q 是 strict core / fallback reference

R1-T14 的实际结果显示，qT=.25 与 qV=.30 带来 material 的状态覆盖和区间几何变化，并通过相应正式结构门禁。因此，四个 q-vector center 作为 R2 的 primary freeze candidates。

对应 shared-q：

```text
qP=qC=qT=qV=.20
```

在 R2 中承担两种角色：

1. **strict core reference**：当对应 q-vector 被选中时，shared-q 只作为同一 `state_line × W` 内部的严格核心标记，不生成额外独立产品版本。
2. **fallback candidate**：如果对应 q-vector 被拒绝，且 shared-q 的事件几何 sidecar 通过全部 hard gate，T04 可以将 shared-q 选为正式版本。

因此，T03 除四条 primary route 的 36 个 `d×g` cells 外，还必须生成对应 shared-q 的 strict-core / fallback geometry sidecar，保证 T04 的 fallback 决策有正式事件证据。

约束为：

```text
retain_as_strict_core_only
→ 对应 state_line × W 的 q-vector 必须 selected。
```

同一 `state_line × W` 下，默认不得同时将 q-vector 和 shared-q 冻结为两个独立产品版本。若确有必要，必须在用户决策记录中提供明确 override justification，并通过独立科学审阅。

### 3.3 qT=.30 与 qV=.25 的边界

```text
qT=.30：保留为 immediate-neighbor sensitivity evidence，默认不进入 primary T03 shortlist。
qV=.25：R1-T10 已标记 do_not_freeze，R2 不得复活为冻结候选。
```

qT=.30 可以支持 qT=.25 的邻域连续性判断，但不能因 coverage 更高或 fragment 更低自动替代 qT=.25。若 R2 希望将其升级为 primary candidate，必须单独修改 R2-T01 shortlist contract，并重新评估完整 T03/T04 family；不得在看到 T03 结果后临时加入。

### 3.4 W120 与 W250 是独立窗口路线

W120 与 W250 使用不同的历史参考尺度，R2 将其治理为：

```text
W120 track：short-reference state track
W250 track：long-reference state track
```

这些名称只描述历史参考窗口，不预先宣称交易周期、持有周期或 release 速度。

同一状态线的两个窗口：

```text
不要求存在集合包含关系；
不能因 Jaccard 低而淘汰；
不能在冻结包中描述为同一状态的等价实现；
若同时冻结，必须使用不同 state_version_id；
必须向 R3 移交 overlap / intersection 信息。
```

### 3.5 不产品化任何额外 PCT parent 状态

R2 不建立、不冻结、不物化：

```text
S_PCT_parent_depletion_v1
```

或任何为了支撑 `S_PCVT` 而增加的 PCT parent 产品版本。

R1 中的 same-parameter PCT parent 只用于 `V_given_PCT`、nested null 和 parent-child 审计。R2 可引用相应 R1 evidence，但不将其升级为状态产品、event table 或 R3 入口。

`strict_core_member` 必须是**同一状态线内部**的严格阈值标记：

```text
S_PCT qT=.25 的 strict core = 同 W/K 的 S_PCT qT=.20；
S_PCVT qV=.30 的 strict core = 同 W/K 的 S_PCVT qV=.20。
```

它不是 S_PCT / S_PCVT 之间的 parent 产品关系。

### 3.6 `K/d/g` 的固定语义

所有 primary q-vector 与 shared-q reference 均使用 `K=3`。R2 不重新扫描 K。

```text
K：daily state confirmation。
   连续 3 个 raw true 日后，第 3 日成为第一个 confirmed_state=true 日；不得回填前两日。

d：event qualification threshold。
   confirmed interval 中 confirmed_state=true 的交易日数必须 ≥ d，事件才取得资格。

g：event-zone grouping tolerance。
   两个 qualified confirmed intervals 中间最多允许 g 个 eligible 且 confirmed_state=false 的交易日，
   才可归入同一个 retrospective event zone。
```

正式网格为：

```text
d ∈ {1,2,3}
g ∈ {0,1,2}
```

`d` 必须使用 `≥d`，并且只按 confirmed true trading days 计数。下列日不得计入 d：

```text
raw true but not confirmed；
unknown；
blocked；
bridged gap；
calendar non-trading day。
```

`g` 只允许桥接 eligible 且 confirmed false 的交易日。unknown / blocked 是 hard break，不得桥接。

### 3.7 `confirmed_state` 与 `event_zone_member` 的边界

`event_zone_member` 是 retrospective grouping 字段，可能包含 bridged false days，也可能只有在后一个 interval 出现后才能最终确定。

R2→R3 contract 必须规定：

> R3 构建状态暴露风险集、事件锚点和未来标签起算点时，只能使用当时已经可见且 `confirmed_state=true` 的日；不得以 `event_zone_member`、`is_bridged_gap` 或事后完整 zone span 替代 confirmed state。

机器断言至少包括：

```text
risk_set_eligible = true → confirmed_state = true；
is_bridged_gap = true → confirmed_state = false；
event_zone_member = true 不推出 risk_set_eligible = true。
```

`event_id` 和 `event_zone_member` 可以用于聚类、事件去重、区间统计和未来样本重叠治理，但不能扩张状态暴露日。

### 3.8 冻结数量不预设

R2-T04 可能最终选择：

```text
4 个版本；
3 个版本；
2 个版本；
1 个版本；
0 个版本。
```

零冻结版本是合法 R2 结论。此时 R2 可以完成验收，但 R3 保持关闭。

## 4. R2 决策层级

R2 不是新的结构显著性搜索阶段。R2 的正式决策采用四层结构。

### 4.1 Hard gate report

hard gate 由程序自动生成，不允许人工 override。至少检查：

```text
R1 handoff status 合法；
input lineage / manifest / schema / hash 通过；
R1 global null 与 nested null 通过；
年份稳定性和 existence floor 通过；
R1-T10 do_not_freeze / blocked 候选未被复活；
T03 event count、年份覆盖和几何指标达到 T02 预注册门槛；
d/g 具有合理参数响应；
bridge、open event、drop rate 未超过门槛；
无全零、全一、全 NULL、数量级突变或 availability 异常；
strict-core subset 与计数守恒通过；
窗口身份治理约束通过。
```

输出必须逐 candidate / d / g 报告：

```text
gate_id
gate_status
observed_value
threshold_or_contract
failure_reason
source_artifact
```

任何 hard gate fail 的候选不得被用户选中。

### 4.2 Pareto + complexity evaluation

通过 hard gate 的候选进入自动 comparison table。禁止使用未预注册的加权总分。

**Level 1：结构和事件几何 Pareto 比较**

```text
confirmed event coverage；
qualified event count；
fragment / short-interval drop；
duration distribution；
bridged gap ratio；
open event ratio；
year concentration；
R1 affected Delta / Lift；
global / nested null；
V selectivity 与 security heterogeneity；
state identity / overlap。
```

**Level 2：复杂度和治理成本**

```text
changed q component count；
是否携带 selection_path_not_independently_confirmed；
是否有 immediate-neighbor support；
warning 数量和严重度；
是否产生额外生产字段或版本；
两个窗口同时冻结对 R3 的 overlap 处理成本。
```

自动推荐采用字典序：

```text
hard-gate pass
→ Pareto non-dominated
→ material structural / geometry advantage
→ neighborhood support
→ fewer unresolved warnings
→ lower complexity
→ baseline proximity
```

简洁性只作为后置 tie-break，不能否定已经超过预注册 robust envelope 的 material 改善。

### 4.3 User decision record

最终选择由研究者作出，并形成独立、可哈希、不可被自动报告覆盖的决策记录。

每个候选至少记录：

```text
candidate_id
automatic_recommendation
user_disposition
selected_d
selected_g
primary_reason_code
secondary_reason_codes
accepted_warnings
rejected_alternatives
override
override_justification
evidence_refs
reviewer_identity
decision_time
```

允许的最终 disposition：

```text
selected
rejected
retain_as_strict_core_only
```

若人工决定与自动推荐不同，必须：

```text
override = true；
填写 explicit justification；
说明接受的 trade-off；
引用具体 evidence。
```

人工可以 override Pareto 推荐，但不能 override hard-gate failure。

### 4.4 Freeze decision 与 freeze plan

T04 基于自动报告和用户决策自动生成：

```text
r2_t04_freeze_decision.json
r2_t04_freeze_plan_manifest.json
```

T04 只形成 freeze plan，不产生具有最终法律效力的 freeze manifest。最终 freeze manifest 必须等待 T05 实际物化和 T06 回放验收通过后，由 T07 生成。

## 5. R2 工作路线

### R2-0：参数候选收敛与 shortlist registry

R2 首先绑定 R1-T10 final package 和 12 行 decision matrix，生成确定性的候选处置与 shortlist。

Primary shortlist 恰为四条 q-vector route。T01 同时登记：

```text
4 个 shared-q strict-core / fallback references；
2 个 qT=.30 sensitivity-only neighbors；
2 个 qV=.25 excluded rows。
```

T01 不运行 `d/g`，不作最终选择，不冻结版本。

### R2-1：冻结事件语义、指标定义与 hard gate

T02 必须在 T03 正式运行前冻结：

```text
K/d/g 语义；
d ≥ threshold 的精确定义；
g bridge 和 hard break；
event qualification / finalization time；
open interval 处理；
confirmed_event_coverage 与 zone_span_coverage；
fragment / drop / bridge / open 指标；
W120/W250 own-sample 与 common-sample denominator；
hard gate 数值门槛；
R3 risk-set confirmed-only 规则；
forbidden outputs；
异常停机和回退规则。
```

具体 numeric hard-gate cutoff 由 R2-T02 task contract 预注册。本阶段总纲不事后替 T02 选择数值。

门槛可以按 `S_PCT` 与 `S_PCVT` 分别设定，但不得按 W120 / W250 单独定制，以免对窗口结果过拟合。

### R2-2：四路线 `d×g` 事件区间几何扫描

四条 primary route 分别执行：

```text
d={1,2,3} × g={0,1,2}
```

共 36 个 primary cells。

对应 shared-q 必须生成 strict-core / fallback geometry sidecar，但不得因此默认产生独立冻结版本。

每个 cell 至少输出：

```text
confirmed_event_coverage
zone_span_coverage
qualified_event_count
qualified_confirmed_day_count
event duration mean / median / quantiles
upstream_singleton_interval_rate
short_interval_drop_rate
post-merge short-zone rate
bridged_gap_count
bridged_day_ratio
merge_ratio
open_event_count
open_event_ratio
events_per_security
events_per_year
nonzero_years
max_year_share
overlapping_event_count
unqualified_interval_count
own_eligible denominator
common_W120_W250 denominator
```

T03 还必须输出同一状态线 W120/W250 的：

```text
intersection days
W120-only days
W250-only days
Jaccard
matched / overlapping event count
```

低 Jaccard 不是 hard failure，只表明窗口状态身份不同。

正式运行后必须读取实际 artifacts，独立复算关键 count 和 rate，并扫描：

```text
全零 / 全一 / 全 NULL；
d 或 g 参数无响应；
event count 爆炸或坍缩；
bridge 主导；
持续期数量级突变；
与上游 confirmed interval 不守恒；
W availability denominator 不一致。
```

完成 runner 和 validator 不代表 T03 完成。异常未解释前，不得推进 T04。

### R2-3：Hard gate、Pareto 推荐、用户决策与 freeze plan

T04 是 R2 的最终参数和事件规则决策任务。其输入为：

```text
R1-T10 candidate matrix；
R1 structure / warning evidence；
R2-T01 shortlist；
R2-T02 contract；
R2-T03 actual event geometry artifacts。
```

T04 按 §4 的四层结构处理每个 q-vector route、对应 strict core 和可选 fallback。

窗口独立性治理检查要求：

```text
不得因 Jaccard 低淘汰；
不得声称 W120 / W250 等价或互相包含；
若同时 selected，必须分配不同 state_version_id；
必须记录 R3 overlap handling requirement。
```

T04 输出 freeze plan，不生成 canonical row artifacts。

### R2-4：selected-only canonical 物化

T05 只读取通过 T04 final gate 的：

```text
r2_t04_freeze_decision.json
```

并且只物化：

```text
disposition = selected
```

的状态版本。物化数量不预设。

若 selected count = 0：

```text

生成合法 no_frozen_version result package；
后续 T06/T07 执行 no-version closure；
```

T05 不得：

```text

修改 W/q/K/d/g；
```

### R2-5：canonical 回放、无前视与一致性验收

```text

zone_finalization_time 未提前；
unknown / blocked 未被 bridge；

manifest、schema、hash、lineage 可复现。
```

T06 必须提交正式 result analysis 和 anomaly scan。工程 validator passed 但实际结果出现退化输出时，任务不得完成。


T07 不得对未通过 T06 的版本生成 frozen 状态。

### R2-7：R2 验收与 R3 交接

T08 验收 R2 是否形成合法冻结结果和唯一 R3 入口。

若 frozen version count ≥ 1，R3 handoff 至少包含：

```text

```

若 frozen version count = 0：

```text

```

## 6. 候选与版本处置定义

### 6.1 `selected`

候选通过全部 hard gate，并经自动推荐、用户决策和独立审阅后被纳入 freeze plan。T05 必须物化，T06 必须验收，最终是否 frozen 取决于实际 artifact 是否通过 T06。

### 6.2 `rejected`

候选因 hard gate fail、Pareto / complexity 劣势、不可接受 warning 或用户明确拒绝而不进入 T05。必须保留 primary reason、evidence 和替代候选，不得从 registry 删除历史记录。

### 6.3 `retain_as_strict_core_only`

shared-q 不作为独立产品版本，只作为被选 q-vector 同状态线、同 W/K 下的严格核心标记。

必须满足：

```text
对应 q-vector disposition = selected；
shared-q 是 q-vector 的严格子集；
subset / count / identity 守恒通过；
T05 只写 strict_core_member 字段，不生成额外 state_version_id 或 event_id。
```

### 6.4 `needs_revision / blocked`

它不是最终候选 disposition，而是任务或 freeze plan 的失败状态。任何 unresolved anomaly、lineage failure、hard gate contract 缺失或用户决策不完整都应阻止下游推进。

## 7. R2 验收条件

R2 的最低验收门槛如下：

1. **输入门槛**：R1-T10 final gate、12 行矩阵、R0-T10 / R0-T15 lineage、schema、hash 全部闭合。
2. **shortlist 门槛**：四条 primary q-vector route、shared-q strict-core / fallback、qT=.30 sensitivity 和 qV=.25 exclusion 均有确定角色。
3. **协议门槛**：K/d/g、`≥d`、g bridge、可用时点、可比指标、hard gates 和 R3 risk-set guard 在 T03 前冻结。
4. **事件几何门槛**：四条路线完成 36-cell primary scan，shared-q sidecar 完整，实际结果通过合理性分析和异常扫描。
5. **决策门槛**：自动 hard gate、Pareto / complexity、用户决策和 freeze plan 完整；hard gate 不能被人工覆盖。
6. **物化门槛**：T05 只物化 selected 版本，数量与 T04 decision 完全一致。
7. **无前视门槛**：confirmation、qualification、finalization、bridge 和 risk-set 字段全部通过回放。
8. **冻结门槛**：最终 freeze manifest 只能引用 T06 已验收的实际 artifacts。
9. **审阅门槛**：formal task 的实际结果分析、anomaly scan、独立科学审阅和 repository final gate 完成。
10. **零版本边界**：允许零版本冻结；此时 R2 可关闭，但不得放行 R3。

## 10. 必须 task 与可选扩展的边界

R2-T01 至 R2-T08 是 R2 的最低必要路线。当前不单列额外可选研究 task，避免在冻结阶段继续扩展候选空间。

以下情形必须新建独立 task 或退回上游，而不能在既有 R2 task 中临时扩大范围：


回退原则：

```text

```

## 11. R2 到 R3 的交接原则

R2 交给 R3 的不是“交易信号”，而是具有唯一版本、明确可用时点和严格风险集边界的状态与事件接口。

R3 必须遵守：

```text
只读取 R2 final freeze manifest 登记的版本；
只用 confirmed_state=true 日构造状态暴露和风险集；
不得以 event_zone_member 或 bridged gap 扩张状态日；
W120 / W250 是独立窗口路线，不预设交易周期；
若两个窗口同时冻结，必须显式处理同证券同日期和事件区间重叠；
strict_core_member 只能作为同一冻结版本内部的严格度分层；
selection_path_not_independently_confirmed 必须进入 R3 limitation；
不得读取 rejected、sensitivity-only 或未物化候选。
```

R3 才能定义和冻结：

```text
release；
future label；
risk-set exclusion；
matched control；
outcome horizon；
precision / recall；
与 outcome window 有关的 cooldown。
```

R2 的最终含义是：

> R1 证明候选状态具备结构资格；R2 决定哪些参数和事件规则可以被冻结，并把被选版本物化成可无前视消费的 canonical 状态与事件接口；R3 再研究这些冻结状态之后发生什么。
