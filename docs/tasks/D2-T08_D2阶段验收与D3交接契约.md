## 目标

- 建立 D2 acceptance 与 D3 handoff 的 contract。
- 明确 D3 未来只能接收治理后的 raw price refs、continuous price refs、membership refs、calendar refs、trading constraint refs、quality flags、mechanical gap flags、PCVT input readiness flags、source snapshot refs 和 observed_at/revision policy。
- 明确 R0 及后续阶段不得绕过 D3 直接读取 D1/D2 raw tables。

## 非目标

- 不把 BAOSTOCK、HITHINK 或任何候选源升级为 formal source。
- 不生成 `d1.raw_market_prices`、`d2.adjusted_market_prices` 或 D3 `daily_market_observations`。
- 不创建 run manifest、dataset manifest 或 source snapshot manifest。
- 不计算 PCVT、R0 state、returns、labels、future outcome、breakout、backtest 或 portfolio。

## D2 Acceptance Gates

- `membership_alignment_accepted`：D2 membership alignment 已存在，800 成分计数保留，D3 不读取 G0 raw evidence，只使用 `membership_ref`。
- `raw_price_layer_gate`：未来 formal raw layer 必须具备 `data_version`、`universe_id`、`time_segment_id`、`security_id`、`trading_date`、raw OHLCV、`amount`、`trading_status`、`price_limit_status`、`source_registry_id`、`source_snapshot_id`、`observed_at` 和 `run_id`。
- `continuous_price_layer_gate`：未来 continuous price layer 必须具备 adjusted OHLC、`adjustment_factor`、`adjustment_method`、`factor_as_of_time`、`corporate_action_flag`、`adjustment_revision`、source refs 和 `run_id`。
- `quality_flags_gate`：未来 D2 quality flags 至少覆盖 raw OHLCV、continuous OHLC、raw/continuous reconciliation、amount/volume unit、DailyVWAP range、trading constraint、price limit、mechanical gap 和 PCVT input readiness。
- `source_snapshot_gate`：formal D3 component refs 必须解析到 source registry 和 source snapshot；D3 不嵌入 raw response bytes 或 vendor export。
- `observed_at_gate`：每个 D3 observation 必须有 `observed_at`；final-revised history 不得声称 point-in-time；unknown observed_at 阻塞 formal D3。
- `no_future_information_gate`：D3/R0 不得包含或依赖 future return、label、breakout direction、outcome、target、backtest 或 portfolio 字段。
- `trading_constraint_gate`：`unknown` 不得静默转 normal/none；停牌、零量和涨跌停必须保留诊断标记。
- `mechanical_gap_gate`：公司行为机械缺口不得当作市场缺口；unknown gap attribution 不得当作 none。
- `pcvt_dependency_gate`：PCVT 仍为 proposed；D2-T08 不生成 PCVT values；R0 只能在 D3 contract accepted 后计算指标。

## D3 Observed_at / Revision Policy

- If `history_revision_class = final_revised_history`, D3 cannot claim point-in-time support。
- If `factor_as_of_time` is missing, formal continuous price use is blocked。
- If `revision_timestamp` is missing, formal revision-aware use is blocked。
- If source terms are pending, formal ingestion remains blocked。
- Exploration-only source can support exploration readiness flags but not formal D3 facts。
- D3 must propagate `research_use_tier`。

## R0 No-Bypass Policy

R0 and later stages must not read D1/D2 raw tables directly once D3 is introduced. R0 may only read `d3.daily_market_observations` or later D3 views. D3 must reference D1/D2 component artifacts by stable refs and must not embed vendor raw payload or row-level source extracts.
