# R0-T11 R0 审计报告与 R1 交接

R0-T11 是 R0 阶段最后一个收口任务。它不改变 R0 状态定义、指标定义、参数网格或事件规则，只消费已经合并的 R0-T01 至 R0-T10-05 task 文档、formal evidence、manifest summary 和 validator output，形成可审计、可追溯、可供 R1-T01 消费的 handoff package。

本任务不读取原始行情源，不绕过 R0-T10 evidence chain，不重新计算 R0-T04、R0-T05、R0-T06、R0-T07，不重新运行 27 组 full-grid，也不生成 future label、future return、release direction、backtest、portfolio 或 trade signal。R0-T11 的输出仅限 audit report、R1 handoff、evidence index、known limitations、audit validator 和 R0-T11 evidence。

## 输入

R0-T11 的正式输入是已经合并到 main 的 R0 task 文档和以下 formal evidence：R0-T10-01 R0-T04 raw metrics evidence、R0-T10-02 R0-T05 strict-past score evidence、R0-T10-03 R0-T06 nested state evidence、R0-T10-04 R0-T07 confirmation / interval evidence、R0-T10-05 authorized input manifest and full-grid evidence。

R0-T11 必须确认这些 evidence 均为 completed，R0-T10-05 validator_status 为 passed，R0-T10-05 记录 `R0-T11_allowed_to_start=true`，27 config 全部 completed，failed config count 为 0，并且 source evidence hash、input artifact hash、synthetic marker 与 raw/external lineage 检查均 passed。R0-T07 和 R0-T10-05 的 confirmed interval 为 0 必须作为真实分布记录，不得解释为运行失败。

## 输出

本任务新增 `docs/reports/r0/R0_audit_report.md`、`docs/reports/r0/R0_r1_handoff.md`、`docs/reports/r0/R0_evidence_index.md` 和 `docs/reports/r0/R0_known_limitations.md`。本任务同时新增 `src/r0/r0_t11_audit_validator.py`、`src/r0/r0_t11_audit_validator_cli.py`、thin wrapper `scripts/r0/validate_r0_t11_audit.py`、R0-T11 evidence 以及对应测试。

这些输出不得包含行级 payload，不得复制 generated DuckDB、Parquet、CSV 或 JSONL 内容。报告只记录路径、hash、row count、coverage、gate 和审计结论。

## 验收标准

R0-T11 audit validator 必须 passed。validator 必须确认 required report files 和 formal evidence files 存在，formal evidence gates 已打开，audit report 包含 R0 completed / R1 allowed / zero interval explanation / 27 config grid 等必要内容，并且报告和 handoff 中没有肯定式宣称 strategy validated、trade signal ready、backtest passed、future return analyzed、parameter optimized 或 R1 completed。

只有 audit validator passed 且 R0-T11 evidence completed 后，`docs/tasks/README.md` 才能推进到 `current_stage: R1`、`current_task: R1-T01 状态存在性与频率轮廓`、`next_planned_task: R1-T02 结构关系与协同约束检验`，并把 `R0-T11` 标记为 `completed via PR #74`。在此之前不得推进 R1。

## 失败状态

若任一 formal evidence 缺失、未 completed、validator 未 passed、R0-T10-05 gate 未打开、报告缺失零区间解释、README 在无 evidence 情况下推进，或报告出现禁止的肯定式交易/回测/参数优化表述，本任务必须保持 blocked 或 failed，不得推进到 R1-T01。
