# D3-T12 开放候选层门禁与下游消费审计解耦

状态：completed via PR #60。

## 目标

本任务修正 D3 candidate generation 与下游 research consumer readiness 的边界。D3-T11 生成的是开放候选层，不等于 formal data version，也不等于 R0 或任一后续研究路线已经具备消费资格。D3 层只负责记录通用质量、evidence、lineage 与 warning；R0-R6 或其他研究路线必须在各自 task 内定义 consumer readiness profile，并在消费前独立检查。

## 非目标

本任务不读取真实数据，不重新运行 D3-T11，不生成或提交 DuckDB、CSV、JSON generated artifact，不发布 formal `data_version`，不计算 PCVT raw values、percentiles、scores、states、state intervals、future labels、returns、backtest 或 portfolio。本任务不修改 D2/D3 字段语义，只修正候选层门禁和下游消费审计的责任划分。

## 输入

输入包括 D3-T11 量额股本换手候选物化 contract、D3-T10 字段可用性口径、D3-T08 registry 边界、D3-T07 candidate observation 语义和 R0-T02 readiness gate。PR #60 的 D3-T11 full-run 摘要以 canonical local output-dir `data/generated/d3/d3_t11_volume_amount_share_turnover_candidate/` 为准；retry-patched artifact 只作为本地备份或审计参考，不作为最终摘要来源。

## 输出

输出包括 `configs/d3/d3_t12_open_candidate_gate_contract.v1.json`、对应 schema、合成测试和 README 状态更新。该任务只提交契约、schema、测试和文档，不提交 generated DuckDB、CSV、JSON、provider raw payload、token、cookie、credential 或 formal manifest。

## 契约与门禁语义

`configs/d3/d3_t12_open_candidate_gate_contract.v1.json` 明确 `policy_evidence_pending_hash` 是 D3 candidate warning，不是 D3 candidate hard blocker。formal release gate 和下游 research consumer gate 后续仍可严格阻塞消费，但该阻塞属于对应 release 或 research task 的消费门禁，不应回写成 D3 candidate generation 的硬失败。

D3-T12 同时废弃把 D3 candidate 行直接标成 `r0_ready_candidate` 的做法。候选生成可以在 `accepted_candidate_with_warnings` 语义下开放给后续审计，但 R-stage readiness 必须由 R-stage task 自己计算并留痕。任何 downstream task 若需要消费 D3 candidate，都必须声明自己的 required fields、allowed lineage、quality gates、unknown/blocked reason vocabulary 和 forbidden outputs。

## 验收标准

验收标准包括：contract JSON 通过 schema；D3 candidate generation 与 consumer readiness 解耦；`policy_evidence_pending_hash` 降级为 candidate warning 的语义被测试覆盖；`r0_ready_candidate` 不再作为 D3 输出或 R0 readiness 依据；README 标记 D3-T12 completed via PR #60，并将后续任务推进到 R0。

## 失败状态

若 schema 不通过、仍把 D3 warning 当作 hard blocker、仍在 D3 输出中定义 R-stage readiness、仍依赖 deprecated `r0_ready_candidate`、提交真实 generated artifact，或用文档说明追认未经门禁的正式数据版本，则本任务失败。

若 D3-T12 语义需要回退，应恢复 `configs/d3/d3_t12_open_candidate_gate_contract.v1.json`、对应 schema、测试和 README 中的 D3-T12 状态，并重新开启单独 PR 修正候选层语义。不得通过覆盖 generated artifact、补写 manifest 或重命名字段，将未通过 release 或 consumer gate 的结果追认为正式证据。
