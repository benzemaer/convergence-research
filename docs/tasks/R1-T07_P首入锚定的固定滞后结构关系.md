# R1-T07 P 首入锚定的固定滞后结构关系

`task_id`: R1-T07

`protocol_version`: R1.v0.3.R1-T07.v1

## 目标

本 task 在 R1-T06 final-gate 已完成并合并后启动，只研究 P 维度从合法非活跃状态首次进入活跃状态后的固定滞后描述性结构关系。固定滞后为第 1、3、5、10、20 个后续观测交易日，目标状态为 C、T、V、S_PCT、S_PCVT。primary baseline 为共享同一前置风险集 `P_{t-1}=false` 且当前日继续非 P 的 stay-out control。

本 task 只提供 descriptive fixed-lag event study 和 secondary structural evidence。它不证明因果，不构成预测检验，不选择参数，不产生 R2 候选，也不替代 R1-T08 的 global/nested null model。

## 前置 Gate

启动前必须确认 PR #82 已合并至最新 `main`，且 R1-T06 result package 满足 `status=completed`、`scientific_review_status=passed`、`anomaly_resolution_status=passed`、`downstream_gate_allowed=true`、`superseded=false`。R1-T06 final-gate package validator 必须为 `passed`。`docs/tasks/README.md` 必须仍显示当前 task 为 R1-T07、next planned task 为 R1-T08，且 `R1-T07_allowed_to_start=true`、`R1-T08_allowed_to_start=false`、`R2_allowed_to_start=false`。

## 输入

正式输入锁定 R1-T06 final package 引用的 repaired R0 artifacts：`r0_t06_dimension_state_results.duckdb` 和 `r0_t06_nested_daily_state_results.duckdb`，路径、SHA-256、row count、security count、日期范围和 schema 由 `configs/r1/r1_t07_p_onset_fixed_lag_relations.v1.json` 固定。禁止回退读取 MarketDB、`.day`、`data/raw`、`data/external` 或其他未授权源。

## 核心定义

完整日序列键为 `security_id × trading_date × W × q`。所有 `LAG`、`LEAD` 和 P 连续 run 计算必须在完整序列上完成，禁止先过滤 eligible、onset 或 valid rows 后再执行窗口函数。

`P_ONSET = previous_P_valid AND previous_P_raw=false AND current_P_valid AND current_P_raw=true`。`STAY_OUT = previous_P_valid AND previous_P_raw=false AND current_P_valid AND current_P_raw=false`。上一行为 unknown、blocked、NULL、invalid 或不存在时不得认定 onset。target@t+k 使用完整序列 `LEAD(..., k)`，表示第 k 个后续观测行，不是日历日差，也不是 valid rows 上的第 k 行。
