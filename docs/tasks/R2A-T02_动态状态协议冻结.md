# R2A-T02：动态状态协议冻结

## 1. 任务定位与当前状态

R2A-T02 只冻结动态状态协议、请求 schema、规范化与 identity 算法。它不读取正式 Score
DuckDB，不计算真实 daily state 或 interval，不执行 q/K 参数响应研究，也不实现 R2A-T03
evaluator。当前状态为：

```text
R2A-T01_status: completed_accepted
R2A-T02_status: protocol_freeze_candidate_pending_review
R2A-T02_started: true
dynamic_protocol_version_candidate: pcavt_dynamic_state_protocol.v1
bound_score_release_id: pcavt-score-w120-v1-c7e04f11a2cd09aa
protocol_package_status: complete_pending_review
real_score_data_read: false
dynamic_evaluator_implemented: false
dynamic_state_materialized: false
R2A-T02_DONE: absent
R2A-T03_allowed_to_start: false
```

唯一允许绑定的输入是 accepted run `R2A-T01-20260718T103110891Z` 的 canonical Score
release `pcavt-score-w120-v1-c7e04f11a2cd09aa`。Accepted handoff SHA-256 为
`142fa9f022025c6097a2651013a1434dae2eb153dc42da8d6450523571c192ad`。本任务只读取该
handoff 及版本化文本契约，没有打开 Score database、review extract 或任何真实输入 DuckDB。

## 2. 版本与协议包

冻结的候选版本为：

```text
dynamic_protocol_version: pcavt_dynamic_state_protocol.v1
protocol_config_version: r2a_t02_dynamic_state_protocol.v1
request_spec_schema_version: r2a_t02_dynamic_request_spec.v1
canonical_request_schema_version: r2a_t02_dynamic_request.v1
```

协议权威配置位于 `configs/r2a/r2a_t02_dynamic_state_protocol.v1.json`。用户提交的科学参数
对象由 request-spec schema 校验；canonical envelope 由 canonical-request schema 校验并由
程序重新计算完整 hash 与短 ID，不能只通过正则即被接受。

以下语义任一变化都必须创建新的 dynamic protocol version，不得原地修改 v1：维度全集或
顺序、q 参数域或单位、weak delta、complete-case、raw state、validity 优先级、K 参数域、
streak reset、confirmation/no-backfill、interval start/end、termination reasons、request hash
preimage、canonical JSON 规则、request ID 格式、JSON duplicate-key policy、joint non-ready
reason derivation、reason vocabulary/order、上游 reason 传播、Score non-finite 定义、primary
termination derivation 或 expected-observation reason 语义。

## 3. Scientific request spec

Request spec 固定包含且只包含：

```json
{
  "request_schema_version": "r2a_t02_dynamic_request_spec.v1",
  "dynamic_protocol_version": "pcavt_dynamic_state_protocol.v1",
  "score_release_id": "pcavt-score-w120-v1-c7e04f11a2cd09aa",
  "selected_dimensions": ["P", "A", "V"],
  "q_by_dimension": {"P": 1000, "A": 1500, "V": 2500},
  "confirmation_k": 3
}
```

所有字段 required，`additionalProperties=false`，不允许 null。`security_ids`、date range、
`evaluation_as_of`、output path、worker count、machine、timestamp、run ID、cache key 与结果
artifact hash 都属于后续 execution scope，不得进入科学 spec 或 request hash。Request ID
也不得充当运行 ID。

### 3.1 selected_dimensions

维度全集和 canonical order 均为 `P,C,A,V,T`。`selected_dimensions` 是集合语义，允许任何
长度 1..5 的非空子集，包括单维请求；空集合、重复或未知维度均拒绝。输入顺序不是科学差异，
canonicalizer 在拒绝非法值后按上述顺序规范化。合法 selected dimension set 总数为：

```text
2^5 - 1 = 31
```

协议不建立固定 P、PC、PCT、PCVT、PCAVT 嵌套状态线。

### 3.2 q_by_dimension

q 只用整数基点表达：`1000,1500,2000,2500` 分别表示 `0.10,0.15,0.20,0.25`。
`q_by_dimension` key set 必须与 selected dimensions 精确相等；缺 key、额外 key、未选维度
key、默认值、自动补齐、字符串、浮点、boolean、null 或域外整数均拒绝。

每个被选维度独立选择四档 q，不存在同 q、联动或排序约束。给定 S 的组合数为 `4^|S|`，
全部非空集合的 q spec 数为：

```text
Σ C(5,m)×4^m = 5^5 - 1 = 3124
```

协议不预先物化这 3124 种组合。

### 3.3 confirmation_k

`confirmation_k` 必须是 JSON integer 且不得为 boolean，正式域是 `{2,3,4,5,6,7}`。
不得默认、取整、截断或字符串转换。K=1 的数学边界是 `raw_state=true` 当日确认，但 v1
request schema 和 builder 必须拒绝 K=1。q 与 K 合计科学 spec 数为：

```text
3124 × 6 = 18744
```

## 4. Canonicalization 与 identity

所有外部 request spec 和 canonical envelope JSON 必须使用同一个严格 parser。任意嵌套层级的
JSON object 只要出现重复 member name，都以稳定 reason code
`duplicate_json_object_key` 拒绝；检查必须发生在 object pairs 转成普通 dict 之前。`NaN`、
`Infinity`、`-Infinity` 同样拒绝。唯一 key 的标准 JSON 不受影响。Canonical envelope loader
只执行严格 JSON parse 和 `validate_canonical_request`，不读取 Score 数据或调用 evaluator。

Builder 顺序固定为：严格解析外部 JSON；schema 校验原始 spec；拒绝重复或未知维度；按
canonical order 排序；
验证 q key set 精确匹配；保留整数基点；构造 canonical spec；计算 hash；派生 ID；构造
canonical envelope。

Canonical JSON 字节语义精确等价于：

```python
json.dumps(
    canonical_spec,
    ensure_ascii=False,
    sort_keys=True,
    separators=(",", ":"),
    allow_nan=False,
).encode("utf-8")
```

Hash preimage 只包含 canonical spec，不包含 request ID、request hash、生成时间、换行或 BOM。

```text
request_hash = SHA-256(canonical spec UTF-8 bytes), lowercase 64 hex
request_id = pcavt-dynreq-v1-<request_hash first 16 hex>
```

完整 hash 是权威 identity，短 ID 只用于展示。如果同一短 ID 对应不同完整 hash，必须返回
`request_id_collision`，不得覆盖、合并或静默接受。

Golden vector 固定为第 3 节示例，其精确 identity 为：

```text
request_hash: fda529f47faece8b7756159247e3c07c261721c4013270981929a4dfe5f9959d
request_id: pcavt-dynreq-v1-fda529f47faece8b
```

Canonical envelope 版本为 `r2a_t02_dynamic_request.v1`，包含且只包含
`request_schema_version`、`request_id`、`request_hash` 和 `spec`。

## 5. Dimension active 与 complete-case joint state

协议常量为：

```text
weak_delta_bp = 1000
weak_delta = 0.10
floating_comparison_epsilon = 1e-12
```

对每个选中维度 D：

```text
qD = q_by_dimension[D] / 10000
main_threshold = 1 - qD
weak_threshold = 1 - qD - 0.10

dimension_active(D,t) =
  score_dimension >= main_threshold - 1e-12
  AND score_dimension_min >= weak_threshold - 1e-12
```

维度 ready 的先决条件是 spine expected observation 存在且为 `present`、dimension row 精确
一行、`eligible_dimension=true`、`validity_status=valid`，且 mean/min Score 均 finite。
未选维度完全不参与 readiness、validity、reason codes 或 raw state。

所有 selected dimensions 必须完整检查，禁止因某维已经 false 而短路其他异常：

```text
joint_ready = all selected dimensions ready
joint_ready=false → raw_state=NULL
joint_ready=true  → raw_state=AND(selected dimension_active)
```

每个 selected dimension 必须独立检查全部 readiness 条件，不得短路，也不得对 NULL 做数值
强制转换。`score_non_finite` 的正式定义是 mean 或 min Score 任一为 NULL、NaN、`+Inf` 或
`-Inf`。派生的维度级 reason vocabulary 固定为：

```text
validity_status=blocked             → D:validity_blocked
validity_status=diagnostic_required → D:validity_diagnostic_required
validity_status=unknown             → D:validity_unknown
eligible_dimension is not true      → D:dimension_not_eligible
mean/min Score NULL/NaN/±Inf        → D:score_non_finite
```

同一行的全部违规必须保留。例如 validity=valid、eligible=false 且 mean/min 都为 NULL 时，
joint validity 仍为 valid，但 joint ready=false、raw state=NULL，并同时产生
`D:dimension_not_eligible` 与 `D:score_non_finite`。

Selected dimension row 的上游 `reason_codes` 也必须完整保留并加 `D:` 前缀；上游与派生 reasons
合并后，先在维度内去重并按字典序排列，再按 `P,C,A,V,T` 的 selected canonical order 输出。
未选维度的 validity、readiness、Score 和上游 reason 均隔离。联合 validity 只聚合 selected
dimension validity，优先级为 `blocked > diagnostic_required > unknown > valid`；它与
`joint_ready` 是不同字段，不能合并。

Expected observation 为 missing 或 listing pause 时，分别增加 joint-level reason
`expected_observation_missing`、`expected_observation_listing_pause`。它们不加维度前缀，并置于
所有维度 reasons 之前。

## 6. Streak、confirmation 与 expected observations

连续性只按同一证券相邻 `observation_sequence` 计算，不能压缩日期或跳过 missing / listing
pause：

```text
raw=true  → 前一 raw=true 时 previous_streak+1，否则 1
raw=false → streak=0
raw=NULL  → streak=NULL，并中断此前 streak
```

`confirmation_event=true` 仅发生在 `streak==K`。`confirmed_state=true` 当且仅当 raw=true 且
streak>=K；evaluable 但未确认或 raw=false 时为 false；raw=NULL 时为 NULL。确认日和区间开始日
都是 streak 首次达到 K 的 observation date，禁止回填此前 K−1 日。

Expected observation status 是 `present|missing|listing_pause`。非 present 行不得跳过，必须令
joint ready=false、raw/streak/confirmed 均为 NULL，并中断 streak 或已确认区间。应存在的 spine
或 dimension cardinality 缺失属于输入契约错误，必须 fail closed，不能降级为普通状态结果。

## 7. Interval 与 termination

区间从 confirmation date 开始，不回填。raw=false、raw=NULL、missing 或 listing pause 都会
终止区间；结束日是 termination observation 之前最后一个 confirmed=true 的 observation，
termination observation 不包含在区间内。v1 禁止退出延迟、gap tolerance、d/g、自动合并或
跨 non-ready observation 延续。

输入结束时仍 confirmed 的区间标记 `open_right_censored`，end date 为最后一个 confirmed
true date。Primary termination reason enum 为：

```text
raw_false
expected_observation_missing
expected_observation_listing_pause
selected_dimension_blocked
selected_dimension_diagnostic_required
selected_dimension_unknown
selected_dimension_not_eligible
selected_dimension_score_non_finite
input_end_open_right_censored
```

Primary category 必须从 observation 事实显式派生：expected missing 映射为
`expected_observation_missing`；expected listing pause 映射为
`expected_observation_listing_pause`；否则只要任一 selected validity 为 blocked、diagnostic
required 或 unknown，依次映射为 `selected_dimension_blocked`、
`selected_dimension_diagnostic_required`、`selected_dimension_unknown`；再否则任一维不 eligible
映射为 `selected_dimension_not_eligible`，任一 Score NULL/NaN/±Inf 映射为
`selected_dimension_score_non_finite`，最后 raw=false 映射为 `raw_false`。所有 joint/dimension
详细 reasons 仍须保留，primary 不能替代它们。

输入结束且区间仍 confirmed 时，primary 单独派生为 `input_end_open_right_censored`，不与上述
优先级混用。

## 8. Zero-event 行为

合法 request 可以产生零 confirmed interval，此时 request status 仍为 completed，interval table
是合法零行表，daily dimension/joint state 仍须完整。零事件不自动视为异常。后续 evaluator 与
analyzer 必须区分合理零事件、全 NULL、全 false、参数无响应、evaluable coverage 退化以及
max raw streak<K；T02 不在真实数据上做这些判断。

## 9. Builder CLI 与实现边界

CLI 为：

```powershell
python scripts/r2a/build_r2a_t02_dynamic_request.py `
  --spec <input-spec.json> `
  --output <canonical-request.json>
```

输入只读；输出父目录必须预先存在；已有输出拒绝；使用原子写入；输出为 UTF-8、无 BOM、LF，
且末尾恰好一个 LF。失败退出非零，成功输出 request ID、完整 hash 与 canonical output path。

`src/r2a/r2a_t02_request_identity.py` 只包含请求 schema 校验、严格类型检查、规范化、identity、
篡改/碰撞拒绝和 canonical envelope 写入，不包含 dimension active、streak、confirmation 或
interval evaluator，也不 import DuckDB。

## 10. 测试与门禁

Synthetic tests 实际遍历并规范化 31 个 selected subsets、3124 个 q spec 和每个 K=2..7
组合形成的 18744 个 q/K spec，但不持久化这些请求。Golden canonical bytes/hash/ID、严格
JSON duplicate-key/non-finite 拒绝、输入顺序不变性、科学参数 identity 变化、严格 q/K 类型、
q key set、envelope 篡改、碰撞、CLI 文本契约与不覆盖均有测试。

协议 truth-table oracle 只存在于 tests，覆盖 main/weak threshold ±1e−12、未选维度隔离、
complete-case、不短路、valid-but-not-eligible、NULL/NaN/±Inf、上游 reason 传播、joint validity
与 readiness 分离、primary priority、raw true/false/NULL、NULL/false reset、K 日确认、
no-backfill、confirmed continuation、missing/listing pause interruption、termination end、open
right censoring、zero interval 与 K=1 数学边界。它不构成生产 evaluator。

## 11. 明确非目标与停止点

T02 不选择最佳 q 或 K，不认为 q=0.10 优于 0.25，不生成任何真实 state/interval，不做 q/K
response，不做真实数据集合单调性审核，不读取正式 Score DuckDB，不实现或启动 T03，不创建
formal run、accepted handoff 或 `DONE`。

当前停止点只能是：

```text
R2A-T02 protocol review
```

在协议候选通过独立审阅前，`R2A-T03_allowed_to_start=false`。
