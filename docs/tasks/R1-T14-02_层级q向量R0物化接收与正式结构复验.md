# R1-T14-02 层级 q-vector R0 物化接收与正式结构复验

`task_id`: R1-T14-02

## 任务定位

R1-T14-02 是 same-sample formal structural revalidation，位于 R0 q-vector 正式物化之后、R1-T10 之前。它只验证已冻结 candidate registry 的结构证据，不选择或冻结最终参数，不直接授权 R2。它不是独立 confirmatory test；T14-01 discovery/selection path 必须作为永久限制保留。

## 触发条件

没有 R1-T14-01 `q_vector_materialization_request`，T14-02 不得启动；T14-01 输出 no-candidate 时，本 task 必须正式记录为 `not_triggered`。存在 request 也只允许先启动单独授权的 R0 materialization，不直接授权 T14-02。T14-01 的 `r1_t14_01_diagnostic_only` artifacts 不得作为本 task 的 formal input。

## 前置 gate

T14-01 final gate 必须通过，且 README 已记录 `R1-T14-01_decision_status: q_vector_materialization_request`、`R0_q_vector_materialization_request_status: fulfilled`、`R0_q_vector_materialization_task_id` 为具体已绑定 R0 task、`R0_q_vector_materialization_status: completed`、`R1-T14-02_status: authorized` 后，T14-02 gate 才可设为 true。R1-T10、R1-T11/T12/T13 和 R2 在此期间保持关闭，除非各 optional task 另有独立触发授权。

## 禁止范围

不得使用任何未来标签，包括未来收益、未来波动、未来方向、未来路径、回测收益或交易结果。不得在本 task 内冻结最终 q-vector、生成可直接生效的 frozen state、按单项最优结果缩小 candidate family，或绕过 R1-T10 直接放行 R2。

## R0 materialization package 契约

必须包含 request identity、candidate registry、strict-past daily dimension/nested state、confirmation、interval、manifest、schema、config hash、输入/output hash、date range、key、lineage、quality/validity 字段、运行环境与绑定的 `R0_q_vector_materialization_task_id`。R0 hash/schema/manifest 不通过时状态固定为 `blocked_return_to_R0`。

## candidate registry 冻结

formal run 前冻结 request 对应 q-vector 与必要邻域、state line、W、K、same-parameter parent、角色和 multiplicity family。不得运行后增删候选或只保留表现较好的向量。本协议 PR 不定义具体 q 值、候选数量或统计 cutoff。该冻结 family 来自 T14-01 提名和 R0 物化后的 same-sample revalidation，不得描述为独立 confirmation sample。

## 邻域稳定性职责

必须检查候选是否为孤立响应、必要邻域内状态身份与结构指标是否连续、availability 是否可比。邻域不稳定必须保留 warning，不得通过删除邻域候选消除。

## 复杂度回报职责

必须比较 q-vector 相对 shared-q baseline 增加的参数复杂度与结构证据变化。复杂度回报是 R1-T10 的 decision input，不是本 task 的 winner score，也不得用单一加权分数替代证据矩阵。

## family-level multiplicity 控制职责

多候选检验必须在 formal run 前预注册 family-level correction 或 max-statistic null，并覆盖完整冻结 T14-02 family。不得只对最终留下的候选选择后校正，不得按最小 p 或最大 JointLift 选 winner。该 correction 只控制 T14-02 formal run 的冻结 family，不校正 T14-01 更大的 one-factor-at-a-time discovery/selection path。

## baseline dominance / Pareto 对照职责

必须逐项比较 shared-q baseline 与 q-vector 在状态身份、availability、层内/层间结构、null、年份、区间和复杂度上的 dominance/Pareto 关系。任何“改善”都必须同时列出退化项与不可比项；不得输出自动 winner。

## 状态身份与 parent-child 约束

必须报告 baseline overlap、新增/丢失状态、same-parameter parent、PCVT/PCT parent-child、confirmation 与 interval 对账。身份漂移过大或 child 超出 parent 为 blocking anomaly。

## 输出 package

正式输出至少包含冻结 registry、R0 lineage reconciliation、baseline identity comparison、intralayer/interlayer/null/year/interval/neighborhood/complexity results、multiplicity result、anomaly scan、result analysis、evidence、engineering validation、author result package、scientific review 和 final-gate validation。结果只进入 R1-T10 matrix。result package、evidence、scientific review 与 R1-T10 handoff 必须永久携带 `selection_path_not_independently_confirmed=true`；该字段不得因 p-value、family correction 或 scientific review passed 而删除。
