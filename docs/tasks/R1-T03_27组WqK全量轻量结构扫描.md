# R1-T03 27 组 W/q/K 全量轻量结构扫描

## 目标

R1-T03 读取 R1-T02 已通过接收审计的 R0 full-grid artifacts，对 27 组 `W/q/K` 候选配置做统一、轻量、可复核的状态空间 profile。输出只描述 raw / confirmed coverage、状态日、证券数、年份覆盖、unknown / blocked、raw fragment、confirmed interval、retention 和相对 baseline 的描述性差异。

## 非目标

本任务不做参数选择，不跑零模型，不计算 `JointLift`、`empirical_p` 或 `z_score`，不生成 R2 decision matrix，不把任何配置标为 `freeze_candidate`、`review_candidate` 或 `do_not_freeze`，不定义 future label，不读取 future return、future volatility、release direction、breakout direction，不做 backtest、portfolio 或 trade signal。

`relative_to_baseline` 只用于同一 `state_name`、同一 metric 下的描述性 ratio / delta，不输出 best、winner 或 optimized config。

## 输入

R1-T03 只能消费 R1-T02 evidence、R1-T02 summary、R1-T02 validation result 和 R0-T10-05 full-grid manifest 中登记的 per-config daily / interval artifacts。不得绕过 R1-T02 读取 raw / external / MarketDB / `.day`，不得读取 loose D3 generated outputs。

## 输出

正式输出写入 `data/generated/r1/r1_t03/<run_id>/`。大型 row artifacts 不提交；小型 governance summary、validation result、profile JSON/CSV 可提交，用于 evidence 在干净 checkout 下复核。

## 并发

formal 默认 `max_workers = 3`，`duckdb_threads_per_worker = 1`，并发层级仅限 config-level。CLI 只允许 `--max-workers 1/2/3`；formal config 和 validator 必须阻止更高并发。parent process 不持有 row payload，worker 只返回小型 profile dict。

## 验收

R1-T03 完成必须满足：27 组配置全部进入 profile；每组输出 `S_P`、`S_PC`、`S_PCT`、`S_PCVT` 四条状态线；若 confirmed interval 为 0，必须标记为 `zero_confirmed_input_fact`，若输入包 confirmed interval 非零，则必须按状态线生成 computed / no-confirmed-for-state profile，不得回填 confirmed state、不得改 K、不得重跑 R0；validator 通过后只允许进入 R1-T04，R1-T07 与 R2 继续 blocked。
