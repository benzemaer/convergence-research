# R2-T08 R2 阶段验收与 R3 交接

## 任务边界

R2-T08 是 R2 的最后一个 PR 级验收任务。它只消费 PR #99 合并后提交的 T07 final freeze manifest、四个 core registry 和已提交 validation artifacts；不重跑 T01–T07，不打开 T05 canonical DuckDB，不重算 daily/event/membership，不重新选择 W/K/q/d/g，不定义 release、不生成 future label、不开展收益或回测研究。

T07 的唯一上游入口为：

```text
data/generated/r2/r2_t07/R2-T07-20260714T034053Z/r2_final_freeze_manifest.json
```

其 committed-byte SHA-256 为 `3a2aea336a535cfe2483a484c0c30c87f8ceb4d03334d4b205674ce30d574a7a`，T07 merge commit 为 `90aba54a54474185fa258afd605e24934bd9a864`。T08 必须从 Git committed blobs 读取并验证输入；工作树替代文件不得成为正式 lineage source。

## 验收目标

T08 必须实际计算并生成 8 个 acceptance gates，8/8 passed 且总 mismatch 为 0：T07 package integrity、frozen version identity、canonical interface binding、registry/policy closure、warning/limitation preservation、window/strict-core closure、release-anchor obligation 和 unique R3 entrypoint。任一 gate 失败即 fail closed，并保持 `R3_allowed_to_start=false`。

冻结接口恰好包含两个版本：

```text
r2_s_pct_W120_K3_qP20_qC20_qT25_qV20_d2_g1_v8
r2_s_pcvt_W120_K3_qP20_qC20_qT20_qV30_d2_g1_v8
```

两者均为 W120/K3/d2/g1。W250、shared-q 独立版本、PCT parent 和其他版本均不得进入 handoff。strict-core 仅是版本内部严格度分层，不是独立 state version、event namespace 或 parent product。

## R3 handoff contract

`r2_t08_r3_handoff_manifest.json` 是唯一 R3 入口，必须满足：

```text
lifecycle=author_stage_pending_scientific_review_and_merge
activation_mode=merged_pr_direct_binding
unique_r3_entrypoint=true
alternative_entrypoints=[]
r3_handoff_eligible=true
R3_allowed_to_start=false
```

release anchor 由 R3 选择。T08 输出三个合法候选，`selected_anchor=null`，不得推荐候选，不得使用 `trigger_trade_date`、quality break 或 right censor 作为 release anchor。

R3 必须消费 T05 canonical tables 并先验证 database SHA-256；不得以同名数据库、T06 replay tables 或 event-zone membership 替代 canonical risk-set fields。confirmed-state exposure 只能使用 `state_risk_set_eligible`，qualified-event exposure 只能使用 daily 或 membership 的 `qualified_event_risk_set_eligible`。

## Formal outputs

正式 run 目录为 `data/generated/r2/r2_t08/R2-T08-<UTC timestamp>/`，核心文件为 acceptance matrix、final acceptance、R3 handoff、frozen window matrix 和 release-anchor obligation；supporting files 包含 input binding、canonical binding、independent validation、anomaly scan、result analysis、output manifest、result package、author-stage scientific review 和 committed-artifact validation。

所有 JSON schema 为 Draft 2020-12 且顶层和 nested objects 均封闭；CSV 使用封闭 header。独立 validator 不得 import generator，必须直接读取 T08 artifacts 和 T07 committed artifacts，输出 12 个 numeric checks，failure_count 为其总和。formal author-stage 固定保持：

```text
status=passed_author_stage_pending_scientific_review_and_merge
r2_evidence_chain_passed=true
r3_handoff_eligible=true
formal_task_completed=false
R3_allowed_to_start=false
```

## 失败与下游边界

旧 run、dirty 工作树、缺失或漂移的 T07 blob、非零 validation、额外 entrypoint、预先选择 anchor、strict-core 产品化或任何 downstream gate 改写均 fail closed。T08 科学 PASS 且 PR 合并后，R3-T01 才能通过 `merged_pr_direct_binding` 验证 exact head、ancestry 和 committed artifacts，并激活 `R3_allowed_to_start=true`；T08 author package 不得自行设置该值。
