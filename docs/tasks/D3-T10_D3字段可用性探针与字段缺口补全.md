# D3-T10 D3 字段可用性探针与字段缺口补全

## 状态

in_progress。

## 目标

D3-T10 用于确认 D3 通用观测层是否具备 R0 八项候选指标和 R0-T02 readiness gate 所需字段，并把量额单位、DailyVWAP、股本、换手率、涨跌停状态和公司行为可比性字段补入 D3 通用字段契约。该 task 放在 D3，是因为这些字段属于跨研究路线复用的市场观测字段，不属于 R0 专属 metric engine。

本 task 同时定义最小 `tnskhdata` 字段探针。`daily` 提供 `vol`、`amount`，其中 `vol` 单位为手、`amount` 单位为千元；`daily_basic` 提供 `total_share`、`float_share`、`free_share`、`turnover_rate`、`turnover_rate_f`、`total_mv`、`circ_mv` 和 `limit_status` 等字段。探针只输出 redacted summary，不提交原始接口返回、token、cookie 或 credential。

## 非目标

本 PR 不计算 PCVT raw metrics，不生成 PCVT percentiles、scores、states 或 intervals，不修改 R0-T01 weak baseline，不直接替换 V1/V2 baseline，不生成 future labels、future returns、breakout direction、backtest 或 portfolio，不发布 formal data_version，不提交真实 tnskhdata 原始数据，也不提交 token、cookie 或 credentials。

## 输入

输入包括 D3-T02 value layer contract、D3-T07 candidate daily observation contract、D3-T08 research dataset registry contract、R0-T01 candidate spec 和 R0-T02 input readiness gate。字段探针可只读检查 D3 candidate DuckDB 或 D3 value layer contract schema；provider 探针仅在本地具备 `TNSKHDATA_TOKEN` 或项目兼容 token 环境变量时执行最小样本调用。

## 输出

输出包括 D3-T10 contract config、JSON Schema、字段可用性探针脚本、tnskhdata 字段探针脚本、标准化纯函数、测试、D3 value contract 更新、README 更新、R0 stage 文档补充和重新生成的合订本。所有输出均为代码、契约或文档，不包含真实 provider raw rows 或 formal data artifacts。

## 字段标准化规则

`volume_raw = daily.vol`，`volume_unit = hand`，`volume_shares = volume_raw * 100`。`amount_raw = daily.amount`，`amount_unit = thousand_yuan`，`amount_yuan = amount_raw * 1000`，`daily_vwap = amount_yuan / volume_shares`。`total_share_raw`、`float_share_raw` 和 `free_share_raw` 来自 `daily_basic`，单位均为万股，并分别乘以 10000 得到 shares 口径。`turnover_float = volume_shares / float_share_shares`，`turnover_free = volume_shares / free_share_shares`。

provider turnover 交叉校验使用 `derived_turnover_rate_pct = turnover_float * 100` 与 `daily_basic.turnover_rate` 比较，并使用 `derived_turnover_rate_f_pct = turnover_free * 100` 与 `daily_basic.turnover_rate_f` 比较。容差只用于字段质量诊断，不用于替换 R0 baseline。

## 质量状态

D3-T10 定义 `amount_volume_unit_status`、`daily_vwap_range_status`、`share_field_status`、`turnover_field_status` 和 `provider_turnover_crosscheck_status`。质量规则包括：成交量和成交额标准化后不得为负；DailyVWAP 应在 raw low/high 容差范围内；总股本、流通股本和自由流通股本应为正，且 `free_share_shares <= float_share_shares <= total_share_shares`；`unknown` unit 不得视为 ready；`zero_volume_flag` 不得解释为低参与度。

## Turnover 边界

D3-T10 补充 `turnover_float`、`turnover_free`、`turnover_rate` 和 `turnover_rate_f` 作为 D3 通用观测字段。R0 baseline 仍为 `V1_VolShrink20_60` 与 `V2_AmountLevel20Pct`。`TurnoverShrink20_60`、`FreeTurnoverShrink20_60`、`TurnoverLevel20Pct` 和 `FreeTurnoverLevel20Pct` 只作为后续 R0-T09 或 R1 sensitivity 的 future alternative V candidate，不在本 PR 中启用。

## Gate 规则

字段探针不得读取 `data/raw`、`data/external`、MarketDB、`.day`、D1/D2 generated outputs 或任意非 D3 边界 DuckDB。tnskhdata 探针不得使用 cookie；token 缺失时必须输出 `blocked_missing_tnskhdata_token` 并正常退出，不得 traceback。任何真实 provider raw payload、credential、PCVT output、future label、return、backtest、portfolio 或 formal data_version 出现在 PR 中，均构成本 task 失败。

## 验收标准

验收需要满足：contract JSON 通过 schema；D3 value contract 明确新增通用量额、股本、换手率和公司行为可比性字段；标准化函数通过合成测试；字段探针能够输出字段矩阵和 indicator schema readiness；tnskhdata 探针无 token 时受控 blocked，有 fake client 时只输出 redacted summary；README 推进到 D3-T10，并保留 R0-T01/R0-T02 completed 与 R0-T03 planned；合订本由脚本重新生成。

## 失败状态

若 contract、schema、tests、README、R0 stage 文档或 config validation 未通过，本 task 失败。若脚本读取 raw/external/MarketDB/.day/D1/D2 真实数据路径、提交 provider 原始返回、提交 token/cookie/credential、计算 PCVT、生成 R0 state、生成 labels/returns/backtest/portfolio 或发布 formal data_version，本 task 失败并应回退。

## 回退方式

回退本 PR 对 D3-T10 task 文档、D3-T10 contract/schema、D3 value contract、探针脚本、标准化函数、测试、`scripts/validate_configs.py`、`docs/tasks/README.md`、R0 stage 文档和合订本的修改即可。由于本 PR 不生成正式数据、不写 DuckDB、不发布 manifest 或 data_version，因此无需数据回滚。
