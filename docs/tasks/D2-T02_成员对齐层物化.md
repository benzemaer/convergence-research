# D2-T02 成员对齐层物化

## 状态

completed via PR TBD

## 目标

本任务将 D1-T04 已提交的 `CSI800_STATIC_2026_06` membership reference
确定性转换为 D2 `d2.membership_alignment` 引用产物。该产物只复制 D1-T04
中已验收的 `security_id`，并补齐 D0 contract 要求的 D2 membership alignment
required fields。

本任务只建立静态研究 cohort 的 D2 对齐引用，不声明历史 point-in-time CSI 800
成分历史，不启动行情、复权、跳空或 D3 生成。

## 非目标

- 不读取 G0 raw evidence。
- 不读取 `data/external`。
- 不调用任何外部 API。
- 不读取 MarketDB 或 `.day` 文件。
- 不采集行情，不生成 `d1.raw_market_prices`。
- 不生成 `d2.adjusted_market_prices` 或 `d2.market_price_quality_flags`。
- 不计算复权因子、raw gap、adjusted gap 或 gap attribution。
- 不写 DuckDB，不创建 committed DuckDB 文件。
- 不创建 run manifest、dataset manifest 或 source snapshot manifest。
- 不修改 D1-T04 membership reference 或 completion report。

## 契约边界

- D2 alignment rows 只能包含 D0 `d2.membership_alignment` required fields。
- `security_id` 必须从 D1-T04 membership reference 的 800 行逐行复制，不得重新生成、
  重新映射或修改。
- `source_registry_id` 固定为 `CSINDEX_OFFICIAL`，仅用于 membership evidence /
  alignment，不得作为 market price、corporate action 或 trading status source。
- `membership_mode` 固定为 `static_cohort`，不得声称历史 point-in-time 成分历史。
- `membership_effective_date` 固定为 `2026-06-12`，并受 DR-001 static-backfill
  边界约束。
- D2-T02 产物可供 D3 membership reference 使用；R0 不得绕过 D3 直接读取 D2。

## 阻塞条件

- D1-T04 membership reference 或 completion report 未通过。
- D2-T01 raw OHLCV source readiness contract 未 accepted。
- alignment row 包含 D0 `d2.membership_alignment` required fields 之外的字段。
- `security_id` 集合与 D1-T04 membership reference 不一致。
- 出现非 `CSINDEX_OFFICIAL` source、非 `static_cohort` mode 或历史成分声明。
- builder 尝试读取 raw evidence、`data/external`、DuckDB、MarketDB、`.day` 或行情文件。
- builder 尝试创建 run manifest、dataset manifest 或 source snapshot manifest。

## 验收标准

- alignment contract、alignment rows 与 report 均通过 JSON Schema。
- alignment rows 精确 800 行，主键唯一。
- 每行只包含 D0 `d2.membership_alignment` required fields。
- `security_id` 集合与 D1-T04 membership reference 完全一致。
- report 记录 `raw_evidence_read=false`、`data_external_read=false`、
  `duckdb_written=false`、`market_data_generated=false`、manifest flags 均为 false。
- builder `--check` 能发现 committed alignment/report 被篡改。
- README 推进到 `current_task: D2-T03`、`next_planned_task: D2-T04`。

## 回退方式

若 schema、builder、alignment rows、report 或测试失败，完整回退本任务新增的 D2-T02
文档、contract、alignment、report、schema、builder、tests 和 README 索引更新。
不得通过修改 D1-T04 reference、读取 raw evidence、手工编辑 800 行、写 DuckDB 或补写说明
来追认失败结果。
