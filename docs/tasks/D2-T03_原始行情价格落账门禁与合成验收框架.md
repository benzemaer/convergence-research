# D2-T03 原始行情价格落账门禁与合成验收框架

## 状态

blocked pending source authorization via PR #27

## 目标

本任务建立 D2-T03 原始行情价格落账的执行门禁、contract、blocking report、validator
和合成验收测试。它固定未来 raw OHLCV 进入 `d1.raw_market_prices` 前必须满足的
字段、主键、来源、snapshot、observed_at、as-of/revision、质量检查和阻塞条件。

当前 D2-T03 仍被 source authorization、raw snapshot、manifest、observed_at 和
revision policy 阻塞。本 PR 不完成真实 raw market prices materialization。

## 非目标

- 不采集行情，不落真实 raw prices。
- 不生成 `d1.raw_market_prices` 正式实体数据。
- 不写 DuckDB，不创建 run manifest、dataset manifest 或 source snapshot manifest。
- 不读取 G0 raw evidence，不读取 `data/external`。
- 不调用外部 API，不读取 MarketDB 或 `.day` 文件。
- 不生成 adjusted prices，不计算复权因子、raw gap、adjusted gap 或 gap attribution。
- 不生成 D3 `daily_market_observations`。
- 不计算 PCVT、状态、事件、标签、收益或回测。

## 契约边界

- 本 PR 只建立 execution gates and synthetic validation。
- `materialization_authorized=false`、`formal_ingestion_authorized=false`、
  `full_market_collection_authorized=false`、`duckdb_write_authorized=false`。
- HiThink / BaoStock 仍为 candidate-only source，不得在本 PR 中升级为正式 ingestion source。
- `CSINDEX_OFFICIAL` 只能作为 membership source，不得作为行情源。
- 供应商 adjusted price、复权价或派生价不得替代 raw trading fact。
- unknown / missing / blocked 状态不得静默转换为 false、0、active、not_at_limit 或 pass。
- D2-T04 不因本 PR 自动解锁；未来若 source snapshot 与授权通过，必须在单独 PR 中执行真实 D2-T03 materialization。

## 阻塞条件

- source terms 或 license / redistribution 未通过正式数据使用审查。
- raw snapshot、SHA-256、source snapshot manifest、dataset manifest 或 run manifest 缺失。
- observed_at 缺失或仅由 trading_date 推导。
- as-of policy、revision policy 或 repeated snapshot comparison 未闭环。
- candidate source 被直接标记为 formal ingestion authorized。
- 尝试全量行情采集、DuckDB 写入或生成真实行情数据产物。

## 验收标准

- contract 与 blocking report 通过 JSON Schema。
- blocking report 明确所有真实数据读取、生成、写入和 manifest flags 均为 false。
- validator 能通过合成合法 rows，并拒绝缺字段、多字段、adjusted fields、错误 source、OHLC 顺序错误、负成交量/成交额、observed_at 缺失或等于 trading_date、非 membership security_id、重复主键和状态静默转换。
- `docs/tasks/README.md` 保持 `current_task: D2-T03`、`next_planned_task: D2-T04`，不推进到 D2-T04。

## 回退方式

若 contract、blocking report、validator、schema、测试或任务索引失败，完整回退本 PR 新增的
D2-T03 文档、contract、blocking report、schema、validator、tests 和 README 索引更新。
不得通过读取真实行情、写 DuckDB、创建 manifest、手工编辑数据或修改上游 D1/D2 产物来追认失败结果。
