# R1-T14-01 层级 q 单变量响应诊断与候选提名

`task_id`: R1-T14-01

## 任务定位

R1-T14-01 是位于 R1-T09 之后、R1-T10 之前的 exploratory structural diagnostic。它诊断 shared-q baseline 是否形成可解释的层级瓶颈，并决定是否值得向 R0 提交少量 q-vector 正式物化请求。它不是参数冻结、正式 q-vector 验收或 R2 handoff。

## 研究问题

本 task 回答单层 q 响应是否改变层级瓶颈、状态身份、结构增量、availability、持续性、碎片和年份稳定性，以及这种响应是否具有连贯 archetype，值得进入后续正式物化。年份稳定、coverage 或 JointLift 单项较高均不足以单独提名。

## 前置 gate

R1-T09 必须 completed、scientific review passed、final gate passed，README 必须明确授权 R1-T14-01。R1-T10、R1-T14-02 与 R2 在本 task 完成前保持关闭。R1-T11、R1-T12、R1-T13 不因本路线自动启动。

## 输入与 lineage 边界

只允许读取版本化 strict-past score、R0 authorized manifest，以及已验收的 R1-T05、T06、T08、T09 evidence。每个输入必须绑定 task/run、path、schema、hash、lineage、日期范围与 gate 状态。禁止绕过 R0 读取 raw/MarketDB，禁止使用无 manifest 的临时候选。

## 允许范围

允许逐层单变量响应诊断、shared baseline 对照、结构响应摘要、candidate archetype 描述和 materialization request/no-candidate 判定。协议只定义职责；具体 q 值、邻域、提名阈值、统计 cutoff 和候选数量必须由后续实现 PR 在运行前冻结。

## 禁止范围

不得进行四维密集 Cartesian search、连续小数优化、未来收益/波动/方向/路径或回测分析。不得按最大 JointLift、最小 p、最大 coverage 或单项排名直接提名。不得物化 q-vector、生成 `freeze_candidate`、直接授权 R1-T14-02、启动 R1-T10 或放行 R2。

## one-factor-at-a-time 原则

每次只改变一个层级 q，其他层保持 shared baseline；必须保留 baseline、变动层、same-parameter parent、eligibility 与 availability 的可比关系。多层联动只能作为候选 archetype 假设，不能在本 task 内作为已验证 q-vector。

## 候选提名职责

提名必须基于多维结构证据与反证，而非单一最优指标。输出只能二选一：

```text
q_vector_materialization_request
no_q_decoupling_candidate
```

request 只授权 R0 对少量 archetype 及必要邻域做正式物化；它不构成 q-vector formal candidate，更不构成冻结建议。no-candidate 必须解释为何 shared-q evidence 足以进入 R1-T10。

## 状态身份与结构响应职责

必须比较 baseline overlap、状态新增/丢失、parent-child、层级瓶颈、retention/Lift/Delta、global/nested association evidence、availability、confirmed interval geometry、fragment、duration 与年份稳定性。状态身份大幅漂移或 parent-child 异常必须作为 material warning 或 blocking anomaly。

## 输出 package 契约

高层输出至少包括：

```text
q_layer_response_diagnostic
q_layer_bottleneck_assessment
baseline_identity_comparison
candidate_archetype_registry
q_vector_materialization_request
no_q_decoupling_candidate
anomaly_scan
result_analysis
evidence
```

materialization request 与 no-candidate 必须互斥。package 必须记录输入 lineage、实际输出 hash、候选/无候选理由、warning、review 状态和下游 gate；不得包含可直接冻结的 q-vector 状态。

## 异常停机条件

输入 hash/schema/manifest 不一致、key 丢失或重复、unknown/blocked 被并入 false、same-parameter parent 错位、参数完全无响应、状态全零/全一/全 NULL、数量级突变、层级不变量失败或 availability 无法解释时必须停止。不得删除不利响应或运行后修改提名规则。

> 完成 runner 和 validator 不代表 task 完成。
> 正式运行后，必须立即读取实际结果 artifacts，完成结果合理性分析和异常扫描，并提交独立 result analysis report。不得只在 PR body 中列出 counts。
> 若出现全零、全 NULL、全一、参数无响应、层级关系异常、数量级突变或与上游 availability 不一致，必须停止下游推进，调查代码、字段契约、lineage、eligibility、validity 和 join 语义。异常未解释前，不得标记 completed，不得推进 README gate。

## author analysis 要求

exploratory 身份不豁免实际 artifact 读取、独立复算、预期与实际对照、替代解释、异常根因和结论边界。validator passed 不能自动生成提名，也不能自动推进 gate。

## 独立审阅与 gate

独立 reviewer 必须复核 one-factor-at-a-time 身份、无未来标签、非单指标最优选择、状态身份和 materialization/no-candidate 判定。只有独立审阅和 final gate 通过后，request 才可交给单独授权的 R0 task；no-candidate 才可将 T14-02 正式记为 `not_triggered` 并推进 R1-T10。R2 始终为 false。

## 完成标准

输入合法、响应诊断完整、实际 artifacts 与 analysis/evidence 已提交、异常已解释、独立审阅通过，并产生且只产生一种合法 decision。README 只能推进到 R0 materialization handoff，或在 no-candidate 路径推进到 R1-T10。

## 失败状态与回退

输入问题标记 `blocked_return_to_upstream`；结果不足或选择规则不合法标记 `needs_revision`；异常未解释标记 `unresolved`。失败不得通过补写文档追认为正式提名。

## 与 R0 materialization 的 handoff

只有 `q_vector_materialization_request` 可触发单独的 R0 task。request 必须携带 archetype 身份、必要邻域理由、输入证据与 hash，但不得在本协议中预置具体 q 值。R0 负责 strict-past 状态、confirmation、interval、manifest、schema、hash 和 lineage。

## 与 R1-T14-02 / R1-T10 的关系

有 request 时，T14-02 仍保持 false，直到 R0 formal package 通过接收 gate；无候选时，T14-02 记为 `not_triggered` 后可进入 R1-T10。T14-01 不得直接授权 T14-02，不得绕过 R1-T10 放行 R2。
