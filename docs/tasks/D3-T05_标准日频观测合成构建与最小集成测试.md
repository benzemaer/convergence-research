# D3-T05 标准日频观测合成构建与最小集成测试

状态：synthetic-only；D3 real generation not authorized。

## 目标

实现 D3 synthetic-only 标准日频观测构建器和最小集成测试，证明 D3-T01、D3-T02、
D3-T03、D3-T04 的 contract 能在同一条 synthetic observation 上闭环。该构建器
只处理 synthetic input payload 或内存对象，不读取真实行情、不连接 DuckDB、不生成正式数据。

## 核心设计

D3-T05 不定义新的正式数据表，不创建 DDL，不生成 `data_version`。它只实现一个
synthetic build path，将 synthetic upstream refs 和 synthetic value fields 合成为：

- `canonical_observation`
- `value_observation`
- `quality_readiness_summary`
- `lineage_validation_result`
- `quality_readiness_contract_check`

`canonical_observation` 必须符合 D3-T01 refs-only canonical table 语义。
`value_observation` 必须符合 D3-T02 value layer 语义。`lineage_validation_result`
必须调用或复用 D3-T03 validator。`quality_readiness_summary` 必须只使用 D3-T04
status/readiness vocabularies，不计算 PCVT values。build output 可以在 tests 中使用
临时目录，但不得写入仓库数据目录、DuckDB、manifest 或 `data_version`。

## 非目标

- 不读取真实 D1/D2/D3 data。
- 不读取 MarketDB、`.day`、`data/raw/` 或 `data/external/`。
- 不调用外部 API。
- 不写 DuckDB。
- 不创建 DDL。
- 不创建 run manifest、dataset manifest、source snapshot manifest 或 `data_version`。
- 不计算 PCVT values。
- 不定义 q、threshold 或 state machine。
- 不生成 returns、labels、future outcome、backtest 或 portfolio。
- 不升级 BAOSTOCK/HITHINK formal source。

## 阻塞条件 / 失败状态

- formal ingestion 未授权。
- D3-T05 不授权 DuckDB write、DDL、真实数据物化、manifest 创建或 `data_version` release。
- D2 formal materialization 未完成前，D3-T07 仍 blocked。
- synthetic payload 缺少 required component refs、`observed_at` 或 `research_use_tier` 时构建失败。
- synthetic payload 或 payload path 包含 `data/raw`、`data/external`、MarketDB、`.duckdb` 或 `.day` 时构建失败。
- synthetic payload 包含 PCVT values、future labels、backtest、portfolio、vendor payload 或 raw/qfq/hfq rows 时构建失败。
- 若 contract、schema、builder、tests、README 或 config validation 未通过，本 PR 失败。
- 若 PR 引入真实数据读取、DuckDB 写入、DDL、manifest、`data_version`、PCVT values、future labels、backtest 或 formal source promotion，本 PR 失败并应回退。

## 验收标准

- contract JSON 通过 JSON Schema。
- builder 能从合法 synthetic payload 构建 canonical observation、value observation、quality readiness summary 和空 lineage errors。
- canonical observation 不包含 raw/adj OHLCV、PCVT values、future labels、backtest 或 vendor payload。
- value observation 包含 `canonical_observation_ref`，且 primary key 与 lineage 字段对齐 canonical observation。
- quality readiness summary 使用 D3-T04 vocabularies，且不包含 PCVT values、state、q threshold、future labels 或 backtest fields。
- README 推进到 D3-T05 / D3-T06，且 D3-T07 和 R0 仍 blocked。

## 回退方式

回退本 PR 新增的 D3-T05 contract、schema、builder、tests、任务文档和 README 阶段索引更新。
不得修改 D0/D1/D2 已 accepted 契约来绕过 D3 阻塞条件。
