# D2-T11 来源状态与复权证据补齐、D2验收与D3交接候选

状态：in_progress via PR TBD

## 目标

主动尝试补齐 D2-T10 暴露出的 source status、trading constraint、复权因子、`factor_as_of_time` 与 `adjustment_revision` 缺口，并基于证据重新评估 D2 acceptance 与 D3 handoff candidate。

## 输入

- D2-T09 raw candidate artifact / quality summary / hash summary。
- D2-T10 adjusted candidate artifact / quality flags / readiness / reconciliation / hash summary。
- HiThink 主源证据、本地 supplemental evidence 或显式授权的 remote fetch summary。
- BAOSTOCK missing-only fallback evidence。
- Tushare missing-only fallback evidence。
- D2-T09 / D2-T10 redacted summaries。

## 输出

- `source_status_evidence_candidate`。
- `factor_evidence_candidate`。
- `source_discrepancy_report.json`。
- `source_status_resolution_candidate_report.json`。
- `factor_status_resolution_candidate_report.json`。
- `d2_acceptance_candidate_report.json`。
- `d3_handoff_candidate_report.json`。
- `d2_t11_gate_decision_summary.json`。
- 可提交的 redacted local execution summary。

本 PR 只允许本地 ignored `data/generated/d2/` candidate artifacts 和 candidate reports。

## 非目标

不提交 raw parquet、generated artifacts、row-level prices、source symbols、security mapping rows、vendor payload、API token、DuckDB、accepted manifests、published data versions、D3 rows、PCVT values、R0 state、labels、returns、backtests 或 portfolio outputs。

## D2-T09 依赖

D2-T09 提供 HiThink raw market prices candidate artifact 与 raw source readiness 边界。D2-T11 不重写 D2-T09 raw facts，不改变 source registry 的 accepted formal source 状态。

## D2-T10 依赖

D2-T10 提供 adjusted price、quality flags、mechanical gap、readiness 与 reconciliation candidate。D2-T11 以 D2-T10 blocking reasons 为补证目标，不通过伪造 resolved 状态关闭门禁。

## HiThink 主源采集设计

HiThink 是 priority 0 primary candidate source。脚本默认不调用远程 API；只有显式 `--enable-remote-fetch` 时才读取 `HITHINK_API_KEY`。API summary 不得包含密钥或 row-level payload。

## BAOSTOCK fallback repair 设计

BAOSTOCK 是 priority 1 fallback，只能补 HiThink missing / unresolved 字段，不得覆盖 HiThink 非空值。

## Tushare fallback repair 设计

Tushare 是 priority 2 fallback，只能补 HiThink 与 BAOSTOCK 后仍 missing / unresolved 的字段，不得静默覆盖更高优先级来源。

## Source Status Evidence 设计

覆盖 `trading_status`、`price_limit_status`、`suspension_status`、`st_status`、涨跌停价格、交易日历状态、observed_at、revision 与 resolution status。`unknown` 不得静默转为 normal / none / false / 0。

## Factor Evidence 设计

覆盖 adjustment factor、`factor_as_of_time`、`adjustment_revision`、factor source、observed_at、revision status、direction 与 point-in-time eligibility。不得把 `source_observed_at`、dump date 或 trading date 当作 `factor_as_of_time`。

## Discrepancy Report 设计

不同来源字段冲突必须进入 discrepancy report，`silent_override_count` 必须为 0。a-stock-data 只能处于 rejected/deprecated，不得作为 active source。

## D2 Acceptance 设计

decision enum 包含 accepted、source blocked、factor blocked、point-in-time blocked、quality blocked、exploration-only 与 rejected。只有所有 required evidence resolved 时，才可输出 `accepted_for_d3_candidate_generation`。

## D3 Handoff Candidate 设计

handoff enum 包含 allowed、blocked 与 blocked gate report only。即使 handoff allowed，本 PR 也不生成 D3 rows；D3-T07 必须重新检查 handoff candidate。

## API / Token Handling

支持 `HITHINK_API_KEY` 与 `TUSHARE_TOKEN` 环境变量，以及 local-only `.env.local`。`scripts/configure_d2_source_env.py` 用 `getpass` 写本地 env file，不打印密钥。`.env` / `.env.*` 已被忽略，`.env.example` 只包含空 key。

## Blocking Rules

- source status unresolved。
- factor evidence unresolved。
- missing `factor_as_of_time`。
- missing `adjustment_revision`。
- point-in-time ineligible。
- source discrepancy unresolved。
- quality unresolved。

## D3-T07 Relationship

D3-T07 remains blocked unless D2-T11 `d3_handoff_decision` explicitly allows D3 candidate generation.

## R0 Boundary

R0 remains blocked. 本 PR 不生成 PCVT values、R0 state、labels、returns、backtests 或 portfolio outputs。

## 验收标准

- contract / schema 通过 config validation。
- env helper 不打印 secret，非交互不等待输入。
- acquisition 遵守 source priority 与 missing-only fallback。
- evaluator 决策与 evidence 一致。
- 若 unresolved，明确 blocked reasons 和 next actions。
- 若 resolved，仅允许 D3-T07 作为下一步 gate，不在本 PR 生成 D3。
- 所有验证命令通过，且不提交 generated artifacts。

## 回退方式

完整回退 D2-T11 contract、schema、scripts、tests、task docs、README、env example 与 redacted summary。不得通过补写文档、提交 raw/generated artifacts、DuckDB、manifest、D3 rows 或 PCVT outputs 追认失败结果。
