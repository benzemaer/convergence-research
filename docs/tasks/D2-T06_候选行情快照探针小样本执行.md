# D2-T06 候选行情快照探针小样本执行

## 状态

small-sample execution pending local authorized environment via PR TBD。

本任务是 PR #30 的 separately authorized probe execution 后续小样本执行框架。当前提交只固化执行计划、红acted 汇总报告、runner 门禁和合成验收测试；默认仓库状态不代表外部 API 已调用，也不代表任何正式 D1/D2/D3 数据已经生成。

## 目标

- 在 D2-T06 contract-only 边界内，定义 BAOSTOCK 候选行情快照小样本执行计划。
- 固定样本证券、样本日期、raw/qfq/hfq 调整模式、raw snapshot 本地目录和 committed redacted report 边界。
- 提供 runner，用于 dry-run readiness 校验和本地授权环境下的小样本探针执行。
- 提供只包含覆盖率、哈希计数、source snapshot 计数、反推因子校验摘要和阻塞原因的红acted 报告。

## 非目标

- 不调用 HITHINK API。
- 不读取 MarketDB 或通达信 `.day` 文件。
- 不读取 G0 raw evidence 或 `data/external`。
- 不采集或提交全市场行情。
- 不写 DuckDB，不生成 `d1.raw_market_prices`、`d2.adjusted_market_prices` 或 D3 `daily_market_observations`。
- 不创建 run manifest、dataset manifest 或 source snapshot manifest。
- 不计算跳空归因、PCVT、状态、事件、标签、收益或回测。
- 不推进 D2-T07 或 D3。

## 契约边界

- 候选执行来源仅限 `BAOSTOCK`。
- `HITHINK_FINANCIAL_API` 在本任务中仍不可执行。
- `formal_ingestion_authorized`、`official_dataset_materialization_authorized`、`duckdb_write_authorized`、`raw_data_commit_authorized` 均必须为 `false`。
- committed report 必须是 redacted summary，不得包含 raw/qfq/hfq 行级价格、供应商原始响应正文或 vendor payload。
- `research_use_tier` 固定为 `exploration_only`。
- 小样本探针不能升级 BAOSTOCK 或任何候选来源的正式 ingestion 资格。

## 阻塞条件

- 未设置本地外部 API 授权环境时，`--execute --source BAOSTOCK` 必须失败。
- 样本证券不在 D2 membership alignment 中时必须失败。
- 缺失 raw snapshot、SHA-256、retrieved_at、observed_at、source snapshot 或 revision/as-of 证据时，不得进入正式 D1/D2 落账。
- factor_as_of_time 和 revision timestamp 未验证前，候选复权价格不得进入正式连续研究价格。

## 验收标准

- execution plan 和 redacted report 均通过 JSON Schema。
- runner 的 dry-run 不调用外部 API、不写 DuckDB、不创建 manifest、不生成正式数据。
- runner 对 `--execute` 有显式环境门禁。
- 测试覆盖样本 membership 校验、BAOSTOCK 代码转换、环境门禁、redacted report、SHA-256 摘要和 implied factor mismatch。
- README 保留 PR #30 contract-only 历史锚点，并标明当前 D2-T06 仍未完成正式行情落账。

## 回退方式

完整回退本 PR 新增的 execution plan、execution report、schema、runner、测试、task 文档和 README 索引更新。不得通过补写文档、导入行情、生成 DuckDB 或创建 manifest 追认失败结果。
