# R1-T09 年份稳定性与状态集中度检查

## 任务定位

R1-T09 是 R1 阶段的基础稳定性 formal experiment。它只检查四个预注册 S_PCT/S_PCVT 候选的年度分布、区间几何、同期层间关系、年份集中度、leave-one-year-out 稳健性以及 W120/W250 年度配对差异，为 R1-T10 提供事实与 warning，不选择或冻结候选。

## 冻结范围

正式 registry 恰好包含 S_PCT 与 S_PCVT 在 W120/W250、q=0.20、K=3 下的四个对象。K2、W500、K5、q=0.10、q=0.30 与 layer-specific q vector 均为 not executed。年份固定为 `YEAR(trading_date)`；2026 是截至 2026-06-30 的部分年份。完整 confirmed interval 固定按 `YEAR(confirmation_date)` 唯一归属，calendar-year clipped geometry 作为独立 sidecar。

## 输入与统计语义

runner 只读取 R0-T10-05 manifest 授权的 dimension state、nested daily state、daily confirmation 与 confirmed interval。年度状态和层间 2x2 从这些授权底表独立重建；R1-T04、R1-T06 与 R1-T08 artifacts 只用于 reconciliation 和继承 pooled null 边界。层间 denominator、same-W parent、ordered three-valued semantics 与 unknown/blocked 处理沿用 R1-T06。

## 预注册状态规则

输入、key、年度分区、守恒、parent-child、上游 reconciliation 或部分年份语义错误均为 `blocked_input_contract`。少于两个 evaluable/nonzero 年份为 `insufficient_year_coverage`。任一年度占 raw/confirmed state days、interval population 或 step child count超过 50% 触发 `single_year_majority_warning`。年度 Delta 同时正负、Lift 同时高低于 1 或 LOYO 方向翻转触发 `year_direction_conflict_warning`。候选级只允许 `year_stability_supported`、`year_stability_supported_with_warning`、`insufficient_year_coverage` 或 `blocked_input_contract`。

## 治理边界

首次 author-draft 必须保持 science pending、downstream false、README 未推进、R1-T10 false、R2 false。T08 pooled null 不解释为逐年 null pass；年份稳定也不解释为预测稳定、交易稳定或因果机制。
