## 4. 强制 smoke 与并行边界

Formal full-universe request concurrency 永远为 1；16 个 request 严格串行，不启动 16 个 Python
workers，也不允许多个进程同时扫描 accepted Score database。单个 request 可使用冻结的 DuckDB
internal threads。Chart rendering 不与 evaluator 同时运行，worker count 不超过 4；首版固定为 1。

Synthetic end-to-end smoke 在 pytest temporary directory 内运行两个 canonical requests、至少三只证券，
同时覆盖 confirmed interval 和 zero-event，并贯通 evaluator、独立 output validator、request/year/
termination metrics、response checks、market rolling context、T+5/T+10/T+20、MFE/MAE、chart sampling、
PNG 与 smoke review schema；不提交 smoke 产物。

Implementation Quality 成功后、authorization commit 前，使用 D05 和由
`sha256(score_release_id + ':' + security_id)` 排序得到的固定四只证券，对 4/8/16 threads 分别运行
完整历史 benchmark。Fingerprint 固定为 `arrow_ipc_fixed_logical_row_chunks.v1`：每张表按完整 primary
key 排序，将任意物理 reader batches 重切为除末块外均为 65,536 行的逻辑 chunks，再基于冻结 schema、
逻辑 chunk ordinal/row count 与 chunk bytes hash 生成 profile。物理 reader batch 数量和大小、thread
count、路径、耗时及文件存储布局均不得进入 fingerprint。

三组输出必须在全部生成且未 cleanup 时，对 4/8、4/16、8/16 三组 pair 的五张表执行 primary-key
FULL OUTER JOIN，并以 `IS DISTINCT FROM` 独立比较全部非主键字段。只有 schema、row count、key set、
逐字段逻辑值与 canonical profile 全部一致，benchmark 才可通过；随后按最快值选择 threads，若最快值
与较低配置差异小于 10%，则选择较低值。Receipt 必须在 passed/blocked 两种状态下均先原子落盘，之后
才能 cleanup；实际逻辑差异时，三份输出 DB 连同哈希与大小移入 local-only failure-evidence 目录保留。
选定 threads 与 benchmark fingerprint 写入 authorization metadata，不进入 request hash，formal 中不得改变。

此前 benchmark 的错误为 `thread_fingerprint_mismatch`，但没有留下 receipt 或逐表逐字段证据；当前仅将

`thread_benchmark_status` 仍为 `blocked_previous_evidence_incomplete`，`duckdb_thread_count` 仍未选择。

## 5. Formal gate 与执行

首个 full-universe request 只有在 implementation/authorization Quality、synthetic smoke、thread
benchmark、real-input smoke、clean worktree、Score/market identity 和三倍 Score 大小的 free-disk gate
全部通过后才能开始。Formal 一旦开始即消费唯一 attempt；失败证据保留，禁止自动重跑、resume、换参数
或删除失败目录。

每个 request 使用 full 800-security scope 和完整 observation history。Accepted evaluator 与独立
validator 通过后，保存五表 row count/fingerprint、request/year/termination metrics、response key
extract、interval path 和 chart context，再删除大型 result DuckDB。四个 chart request 各按冻结 strata
选择 12 个 interval，共 48 PNG；人工 worksheet 字段保持空白。

## 6. 科学与结果边界

硬校验包括 q raw/confirmed subset、q joint-ready equality、K raw equality/confirmed contraction、
dimension ladder contraction、marginal target superset 与 non-target invariance，并要求响应非退化。
Market diagnostics 输出 MA cloud、price-to-cloud、ATR/NATR、range、volume/amount/turnover ratios；未来路径
按证券真实 observation offset 计算 T+5/10/20、MFE、MAE 和 peak/trough time。所有 future metrics 只做
描述，不是预测标签、信号或策略收益。

Formal 成功后只提交不超过 60 MB 的 compact review bundle，不提交 Score/MarketDB、完整 result DB、
audit database 或 interval parquet。最终 recommendation 只能继续 owner visual review 或阻塞；本任务不创建
`DONE`，不允许 R2A-T05。
