# R0-T10 正式物化与全网格执行

状态：in_progress；正在由真实 D3 授权源生成 R0-T04 -> R0-T07 upstream artifacts，并执行 R0-T09 dry-run、baseline 与 27 组 full-grid。

## 目标

R0-T10 补齐 R0-T09 之后仍缺失的正式执行链路：定位真实授权研究输入，生成或接收 R0-T04 至 R0-T07 的正式 upstream result artifacts，生成 R0-T09 `authorized_input_manifest.json`，并在通过 dry-run 与 baseline 单组 materialization 后执行 27 组 full-grid production materialization。第二段代码已解除第一段的固定 full-grid 阻断，但 full-grid 仍必须满足正式 upstream、dry-run 和 baseline 前置条件。

## 非目标

R0-T10 不生成 R0 审计报告，不生成 R1 handoff，不计算未来收益标签、未来波动、突破方向、释放方向、backtest、portfolio、trade signal、gap merge 或 cooldown。R0-T10 也不改变 R0-T04 至 R0-T09 的指标、分位、状态、确认、主网格配置、V1 baseline 或 worker 上限定义。

## 输入

正式输入只能来自真实授权数据生成的 R0-T04 至 R0-T07 upstream artifacts，或调用方显式提供的真实 R0-T04 至 R0-T07 upstream artifacts。当前默认真实源为 D3-T11 `data/generated/d3/d3_t11_volume_amount_share_turnover_candidate_clean_rerun/d3_t11_volume_amount_share_turnover_candidate.duckdb` 与 D3-T07 `data/generated/d3/d3_t07_candidate_daily_observation/d3_t07_candidate_daily_observation.duckdb`，R0-T10 会从这两个 DuckDB 读取观测值和调整价格，依次运行 R0-T04、R0-T05、R0-T06、R0-T07 engine，写出正式 upstream JSON。

禁止把 `tests/fixtures/*`、unit test helper、synthetic fixture、`synthetic_smoke_fixture`、`_contract_grid_payload()`、tmpdir test outputs 或手工伪造 JSON 作为 production formal input。若默认 D3 源缺失、字段无法满足 R0-T04 至 R0-T07 engine、或显式提供的 upstream artifacts 不完整，R0-T10 必须返回 blocked，不得生成 authorized manifest 或 full-grid artifact。

R0-T10 入口为：

```bash
python scripts/r0/run_r0_t10_formal_materialization.py \
  --output-dir data/generated/r0/r0_t10/<run_id> \
  --run-id <run_id> \
  --code-commit <git_sha> \
  --dry-run-r0-t09 \
  --baseline-r0-t09 \
  --full-grid-r0-t09 \
  --max-workers 2
```

如果调用方已经拥有真实 upstream artifacts，也可显式传入：

```bash
python scripts/r0/run_r0_t10_formal_materialization.py \
  --output-dir data/generated/r0/r0_t10/<run_id> \
  --run-id <run_id> \
  --code-commit <git_sha> \
  --r0-t04-input <r0_t04_raw_metric_results.json> \
  --r0-t05-input <r0_t05_score_results.json> \
  --r0-t06-input <r0_t06_nested_daily_state_results.json> \
  --r0-t07-input <r0_t07_confirmation_results.json> \
  --dry-run-r0-t09 \
  --baseline-r0-t09 \
  --full-grid-r0-t09 \
  --max-workers 2
```

`--full-grid-r0-t09` 必须与 `--dry-run-r0-t09` 和 `--baseline-r0-t09` 同时使用。缺少任一前置步骤时返回 blocked，reason 为 `full_grid_requires_dry_run_and_baseline`；真实 D3 源生成失败时返回 `formal_upstream_generation_failed`；显式 upstream 缺失时返回 `formal_upstream_inputs_missing`。

## 输出

R0-T10 的正式输出目录为 `data/generated/r0/r0_t10/<run_id>/`。正式 27 组 full-grid 只有在 authorized manifest、dry-run 和 baseline 均通过后才写入 `r0_t09_full_grid/`。本 PR 不提交任何 `data/generated/r0/r0_t10/**/*.json`、DuckDB、CSV gzip、DONE/FAILED、manifest、logs 或 partial 文件。

## 契约与门禁语义

契约固化在 `configs/r0/r0_t10_formal_materialization_contract.v1.json`，schema 固化在 `schemas/r0/r0_t10_formal_materialization_contract.schema.json`。`real_data_read_authorized=true`、`generated_json_write_authorized=true`、`r0_t09_runner_execution_authorized=true`；phase 1 明确 `r0_t09_full_grid_execution_authorized_in_phase_1=false`，phase 2 明确 `phase_2_full_grid_execution_authorized_after_preconditions=true`。`max_workers_default=2`，`max_workers_upper_bound=2`，超过 2 必须拒绝。

正式 upstream 必须包含 `raw_metric_results`、`indicator_score_results`、`dimension_score_results`、`nested_daily_state_results`、`daily_confirmation_results` 和 `confirmed_interval_results`。R0-T09 builder 继续负责 coverage guard，要求 9 个 `(W,q)`、108 个 `(W,q,K,state_name)`，K 仅为 2/3/5，V1 仍为 `V1_TurnoverShrink20_60` / `TurnoverShrink20_60_raw`。

## 验收标准

验收标准是：缺失正式上游时必须 blocked 且不写 authorized manifest；fixture/synthetic 路径必须被拒；显式合法 upstream 输入可生成 R0-T09 manifest 并通过 dry-run 与 baseline 单组 materialization；full-grid 只有在 dry-run、baseline 和 coverage guard 均通过后才能执行；所有输出不得包含 audit report、R1 handoff、future、return、backtest、portfolio 或 signal 字段。

## 失败状态

默认 D3 源缺失、真实 upstream 生成失败、显式 upstream 输入 JSON 无效、row set 缺失或为空、路径疑似 fixture/synthetic、coverage guard 不通过或 baseline 单组失败时，R0-T10 返回 `status=blocked`，写出 summary。baseline 单组失败时，R0-T10 保留 R0-T09 FAILED/log 供排查，不得继续 full-grid。
