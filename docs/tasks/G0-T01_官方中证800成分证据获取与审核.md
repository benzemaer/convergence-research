# G0-T01：官方中证 800 成分证据获取与审核

> Task ID：G0-T01
> 状态：draft
> 关联决策：DR-001
> 目标门禁：G0
> 当前运行资格：blocked
> 执行人与审核人：待指派，且必须相互独立

## 1. 目标

获取并保存 `CSI800_STATIC_2026_06` 所需的官方成分证据，建立可复核的来源、文档身份、
抓取时间、原始文件和 SHA-256 记录，并由独立审核人决定是否满足 G0 证据门。

## 2. 非目标

- 不运行 D0 数据处理；
- 不创建 DuckDB；
- 不抓取个股行情、公司行为或交易状态；
- 不计算 PCVT 指标、状态、事件、标签或收益；
- 不把非官方来源、搜索结果摘要或供应商转述当作官方成分证据；
- 不因文件格式正确而自动批准 G0。

## 3. 受控来源

只允许使用配置注册的官方身份：

```text
source_registry_id: CSINDEX_OFFICIAL
publisher_name: 中证指数有限公司
official_domain: csindex.com.cn
document_type: index_constituent_list
index_code: 000906
```

下载地址必须使用 HTTPS，主机必须为 `csindex.com.cn` 或其子域。重定向链、最终 URL、
HTTP 响应时间和响应头必须记录。若需要登录、验证码或许可证确认，停止自动化并记录
阻塞，不得绕过访问控制。

## 4. 获取产物

原始证据只保存在只读外部数据层，不提交到 Git：

```text
data/external/source=csindex/index=000906/snapshot=<retrieved_at>/
```

至少保存：

- 官方原始响应或原始下载文件；
- 文档标题、文档 ID、指数代码、精确生效日；
- 来源页面 URL、最终下载 URL、抓取时间及时区；
- 原始文件字节数和 SHA-256；
- 许可证或使用条款引用；
- 获取日志和失败日志；
- source snapshot manifest。

禁止只保存解析后的证券代码列表而丢弃原始响应。

## 5. 验证要求

执行人至少检查：

1. 发布者、官方域、文档类型和指数代码与注册项一致；
2. 文件明确对应中证 800，而不是中证 800 行业、风格或策略子指数；
3. 精确生效日可由官方文档直接支持；
4. 证券数量、代码格式、重复项和缺失项可解释；
5. 原始文件哈希可重复计算；
6. manifest 引用的路径、大小和哈希与文件一致；
7. 未把“2026 年 6 月”文本标签代替正式生效日和版本标识。

任何无法验证的项目都必须保持 `pending` 或 `rejected`，不得转换为 `verified`。

## 6. 独立 G0 审核

审核人不得是证据获取执行人。审核必须针对包含证据 manifest 的固定 Git commit，并记录：

```text
status
gate=G0
reviewed_by
reviewed_at
review_commit
independence_attested=true
```

审核人应独立重新计算原始文件 SHA-256，并核对官方来源、文档身份、生效日、证券数量和
manifest。仅当所有检查通过时，才允许在后续提交中同时设置：

```text
membership_evidence.status=verified
operational_status=eligible_for_d0
g0_review.status=approved
```

## 7. 验收标准

- 官方来源身份和文档身份完整；
- 原始证据文件与 manifest 均存在；
- SHA-256 独立复算一致；
- 精确生效日有官方直接证据；
- 独立审核记录完整并绑定固定 commit；
- 配置 schema、契约测试与 Quality CI 全部通过；
- PR 明确说明 D0 是否获准；默认结论为不获准。

## 8. 失败与回退

若官方文件无法取得、许可证不允许保存、文档身份不明确、哈希不一致或独立审核失败，
则任务保持 blocked，`operational_status` 不变。不得使用第三方近似名单替换官方证据；
任何替代来源必须新建决策记录和任务。
