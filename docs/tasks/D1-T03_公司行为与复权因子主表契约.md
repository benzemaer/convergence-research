# D1-T03 公司行为与复权因子主表契约

## 状态

completed via PR #12

## 目标

本任务实现 `d1.corporate_actions` 与 `d2.adjusted_market_prices` 中复权相关字段的契约/边界设计。
目标是基于 `configs/d0/data_product_contracts.v1.json` 中两张表的契约，明确公司行为字段语义、
复权因子与连续研究价格字段边界、受控词表、来源边界、as-of/revision 阻塞条件，以及未来
loader 必须满足的质量检查。

## 非目标

- 不调用任何外部 API。
- 不新增 loader、API client、ETL worker 或 source-specific ingestion 代码。
- 不导入真实公司行为、分红、送转、配股、复权因子或行情数据。
- 不计算复权价格，不生成 D1/D2 DuckDB 实体数据，不创建 committed DuckDB 文件。
- 不生成 raw snapshot、dataset manifest、run manifest 或 artifact manifest。
- 不改变 D0 source registry 的来源资格结论。

- `configs/d0/source_registry.v1.json`

- 本任务文档与 `docs/tasks/README.md` 索引更新

## 契约边界

- `d1.corporate_actions` 定义公司行为事件事实与机械缺口归因，不定义市场释放信号。
- `d2.adjusted_market_prices` 的复权因子和连续价格字段只在本任务中定义契约，不计算、不落实体数据。
- 除权除息、送转、配股、拆并股等机械缺口不得直接解释为普通市场跳空、趋势变化或释放。
- `factor_as_of_time` 不得使用未来公司行为知识污染历史观察。
- 复权因子必须可按 `data_version`、`source_snapshot_id`、`adjustment_revision` 回溯。
- R0/R 研究只能通过 `d3.daily_market_observations` 使用公司行为和复权引用。
- `HITHINK_FINANCIAL_API` 仍仅为候选来源；`BAOSTOCK` 对 adjusted market prices
  也仅为候选/备选来源，不授权正式 ingestion。
- `CSINDEX_OFFICIAL` 与 `A_STOCK_DATA_RECON` 不得作为公司行为、复权因子或连续价格正式来源。

## 阻塞条件

- source terms 未完成正式数据使用审查。
- as-of、factor_as_of_time 与 revision 规则未闭环。
- raw snapshot、SHA-256、source snapshot manifest 或 run manifest 缺失。
- action_terms、ex_date、factor_as_of_time、adjustment_factor 或 adjustment_method 修正无法追溯。
- 公司行为或复权因子质量为 unknown 且无法追溯。
