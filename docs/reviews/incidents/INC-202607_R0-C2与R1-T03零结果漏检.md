# INC-202607 R0-C2 与 R1-T03 零结果漏检

## 摘要

PR #77 基于修复前的 R0 zero-package 生成 R1-T03 profile，其 `S_PC/S_PCT/S_PCVT=0` 与 `confirmed=0` 结果被错误地作为可继续审阅的 formal evidence。PR #78 修复 R0 C2 readiness alias 与 state-specific validity 后，生成 repaired nonzero package，推翻了 PR #77 的结果。PR #77 已按 supersession 关闭，不合并、不 rebase、不 cherry-pick，其 artifacts 不得作为当前 evidence、参数选择依据或后续 formal input。

## Technical Root Cause

技术根因是上游 R0 package 的 C2 readiness / validity 传播存在缺陷，导致下游 R1-T03 在工程上读到了 hash、manifest、schema 均完整的 package，但该 package 的关键状态空间退化为零结果。这个问题属于上游 package 语义错误，而不是 R1-T03 runner 本身的文件读写错误。

## 为什么工程 validator 会通过

工程 validator 主要复核 path、sha256、schema、row count、forbidden fields、lineage 和 gate 字段。它能证明 artifact 可追溯、可读取、无明显前视字段，但不能自动判断“所有关键状态为 0”是否科学合理，也不能替代对真实 profile 结果的解释和参数响应审查。

## 为什么第一次审阅没有及时阻断

第一次审阅没有强制要求执行代理在 formal run 后立即读取和分析真实结果，也没有要求独立 reviewer 复算关键 count/ratio 或检查 baseline 与 challenger 的参数响应。流程上把 `validator_status=passed` 过度等同于结果可接受，缺少科学结果门禁。

## 错误结论的影响范围

PR #77 的结果不得作为当前 evidence、formal input、参数选择依据、冻结依据、研究结论或 README gate 依据。影响范围限定为该 draft PR 及其未合并 artifacts；PR #78 的 repaired nonzero package 是当前有效替代。

## 纠正措施

关闭 PR #77 并删除远端分支，明确 superseded by PR #78 / merge commit `8694cba4ddbd5a18e43ab18454dfc19cfb9903cd`。后续 R1-T04 只能消费 PR #78 合并后的有效 R1-T03 package，不得 cherry-pick PR #77 的旧 result artifacts。

## GOV-T01 的预防控制

GOV-T01 新增 formal experiment 结果包、anomaly scan、author result analysis、independent scientific review 和 final-gate validator。全零、全 NULL、参数无响应、状态退化、漏斗无法对账、上下游 availability 不一致等情况将成为 hard blocker，且 draft 阶段不得推进 README 或 downstream gate。

## 组织原则

本 incident review 不追责个人，聚焦流程和系统。工程 validator 是必要门禁，但不是科学结论门禁；后续流程必须让 reviewer 审核真实结果而不只审核代码和 hash。
